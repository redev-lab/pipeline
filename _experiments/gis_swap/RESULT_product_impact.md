# backfill 과투자 점검 — 제품 출력 기준 재정의 (2026-06-23)

> _experiments/gis_swap/ 전용, redev/·git 미변경. "환경점수 candidate"가 제품 핵심 출력인지부터 확인.

## #1 B1+ feature importance — ★균형 다축(노후도 모델 아님)
| 피처군 | 기여 |
|---|---|
| 공시지가 | 28.1% |
| **노후도** | **27.4%** |
| 호수밀도 | 22.3% |
| 역세권 | 12.5% |
| 면적 5.2 · 형상 3.3 · 접도율 1.3 | 9.8% |

→ 노후도는 **27%** = 사실상 노후도 모델 아님. ★**national swap이 건드리는 건 노후도(27%)+호수밀도(22%)=49%뿐**.
나머지 51%(공시지가·역세권·면적·형상·접도율)는 parcels·land_price·rail 출처라 **national 건물 swap과 무관**.
(missing-building 필지가 candidate서 추락한 건 이 49%가 동시에 0으로 떨어지기 때문.)

## #2 제품 출력 정의 (코드 확인)
- **닮은동네**(retrieval matches): `/report`의 ★**히어로**(프론트 재설계로 주인공). 동네명+유사도%+진행상태.
- **환경점수**(score_pct): ① report 보조줄("서울 전체 기준 상위 X%") ② **지도 히트맵**(BboxLayer 색점) ③ **스크리너**(/screen, score_pct 랭킹 필터).
- candidate flag: report verdict 분기.
→ 제품은 **둘 다** 쓰되, 프론트 위계상 **닮은동네=히어로 / 환경점수=보조·지도·스크리너**.

## #3 출력 기준 national robust 여부
- **닮은동네(히어로)**: 동작 6클러스터 → **평균 교집합 4.5/5 · 1위 유지 3/4**. backfill 없이도 **대체로 견딤**(군집 집계가 per-parcel 손실 희석). 1클러스터만 1위 뒤바뀜.
- **환경점수 candidate(히트맵·스크리너)**: national 30% 손실, backfill 52% 커버로 **25%만 회복**. → 동작 히트맵·스크리너는 candidate를 **과소검출**(real 노후 candidate가 색·랭킹서 빠짐).

## #4 national + '데이터 부족' 정직표기 절충 — 성립
- 닮은동네(히어로 판정)는 national으로 견딤 → **그대로 노출 가능**.
- 환경점수는 backfill-빈 필지를 **'데이터 부족'으로 표기**(거짓 낮은 점수 대신) → 히트맵서 회색/제외, 스크리너서 제외.
- 즉 **닮은동네는 살리고 환경점수 히트맵의 빈 필지만 정직 처리**하는 절충이 성립.

## ★권고 — (B) 닮은동네 위주 + 현 수준 + 정직표기로 충분
- 제품 핵심 판정 = **닮은동네(히어로)이고 national robust**(4.5/5) → ★**candidate 25% 회복은 부차적, 총괄표제부 backfill은 과투자**.
- national swap을 쓰되: 닮은동네 그대로 + 환경점수는 **현 backfill(동작 52%)로 부분 회복 + 잔여 빈 필지 '데이터 부족' 정직표기**. 더 메꾸기(총괄표제부 멀티오퍼레이션)는 **보류**.
- ★단서(B의 예외): **스크리너(/screen, 구 단위 candidate 랭킹)를 핵심 셀링포인트로 밀 경우**, 동작 등 저커버리지 구의 과소검출이 문제 → 그때만 backfill 추가 가치. 현 위계(닮은동네 히어로)에선 불필요.
