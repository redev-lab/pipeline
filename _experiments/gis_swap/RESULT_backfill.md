## #3 데이터 사실(fact) 계보 — 4유형 우회 논리 (사실관계만, 법적 단정 아님)
- **사용승인일·연면적은 '사실'**이고 원본 소유 = **국토부 건축물대장(세움터/EAIS)**.
- 서울 열린데이터광장 AL_D010(4유형) = 그 대장 + 도로명주소건물을 **통합·재배포한 가공본**.
- national AL_D162/D164(1유형 후보) = 같은 통합 데이터의 행안부 국가 배포본.
- → ★동일한 사용승인일 '사실'을 **건축HUB(국토부 대장 직접)** 에서 가져오면 **서울 4유형 가공본을 경유하지 않음**.
  즉 소스 계보상 4유형을 우회해 같은 값 확보가 *기술적으로* 가능. **단 적법성은 건축HUB API의 이용허락범위로
  결정** — 그게 상업 허용이면 성립, 아니면 불성립. (계보·라이선스 사실만; 법적 판단은 변호사.)

## #4 보충 통합(hybrid) 기술 가능성 — 깨끗함
- national 기본 + **손실 60,931 PNU만** 건축HUB 표제부로 채우는 hybrid 로드.
- ★충돌 없음: 보완 대상은 정의상 **national에 건물 0** → 중복·충돌 없이 **순수 추가**(append). PNU 조인 단순.
- 표제부는 건물(관리번호) 단위 → 시군구+법정동+번지로 PNU 그룹핑, building 행 [pnu·사용승인일·연면적·구조] 형태로
  national 테이블에 concat. building_gis 정제 로직(_parse_approval_year 등) 재사용 가능.
- 유일 주의: 표제부 '대지위치→PNU' 매핑(부속지번 다수 건물). 우리 jibun_index/parcels로 해소 가능.
