# GIS건물 4유형→1유형 전환 트랙 — 종합 (2026-06-23)

> _experiments/gis_swap/ 측정 트랙 종합. 본 파이프라인 적용 계획 포함. 개별 근거는 RESULT_*.md 참조.
> redev/ 본코드는 #1·#2만 이미 반영, 나머지는 계획.

## 1. 목적
건물 데이터를 **서울 열린데이터광장 GIS건물(공공누리 4유형, 상업불가)** → **국가표준 GIS건물(1유형, 상업가능)** 으로
전환 가능한지 검증. 판단기준 = **"유료 사용자·법무 앞에서 버티나"**(값 정합성 + 라이선스 + 제품 품질).

## 2. 단계별 결론 (측정 근거)

### (1) VWorld WFS 시도 → 실패
VWorld WFS 건물 레이어 = `lt_c_spbd`(도로명주소건물)뿐. **사용승인일·연면적 없음**(PNU·층수만). → 노후도·연면적 swap 불가. (RESULT.md)

### (2) national GIS건물 SHP 확보 + 어댑터 매핑 (실증)
AL_D162(일반 340,644) + AL_D164(집합 110,360) = 451,004건, EPSG:5186. 서울 AL_D010과 A-코드 레이아웃 상이 →
어댑터: **A1→PNU · A35→사용승인일 · A24→연면적 · A28→구조 · A39→시군구**. 서울 A13과 같은 PNU 378,592건 대조:
사용승인일 **정확일치 99.9%**·연면적 상관 0.9955·구조 100%·시군구 100%. (RESULT.md)

### (3) 노후도 swap — 안전
공통 489,370 필지 노후도 **100% 동일**(|차| 0.0000%p). 값 정합성 통과. (RESULT.md · #1 본코드 검증)

### (4) 빈폴리곤 필터 — ★본코드 반영(#1)
연면적<=0 AND 사용승인일 없음 = 부속·무허가 더미. building_gis.load_buildings에 AND 필터 추가.
**노후도 불변 + 노이즈 133,891 제거**. (RESULT_metric / building_gis.py 반영)

### (5) 호수밀도 — 법정의 불일치 + 약신호 → 하드게이트 제거 ★본코드 반영(#2)
- 법정 호수밀도(동수, 공동주택 세대층 환산)와 현 raw 폴리곤 카운트 불일치. 격자표본 AUC 0.55~0.63 = 약신호.
- 하드게이트(주택정비형 OR-가지)는 25구 전수에서 **4구역만 구제**(전부 임계 경계값 60~66, national swap 시 전부 탈락).
- → stage1 `elig = old_ok and abut_ok`로 OR-항 제거(호수밀도는 표시·ML 피처로 존치). (RESULT_metric2 / stage1.py 반영)

### (6) 노후도 커버리지 — national의 실주거 누락
national이 서울 대비 건물 35% 적음. aging=0 손실 필지 **83,343(17%)**, 그 중 **실주거 69%·노후 85%**(쓰레기 아님).
동작 등 집중 → 동작 candidate(상위10%) **94% 추락**(per-parcel 환경점수). (RESULT_coverage_impact)

### (7) 건축HUB 표제부 backfill (라이선스 제한없음=1유형)
국토부 건축HUB getBrTitleInfo(15134735). 동작 5,783 PNU 수집 → 사용승인일 **52% 커버**.
hybrid aging → candidate **25% 회복**(부분). 회복 제한 이유 = 48% 잔여 빈 표제부 + 2-hop 이웃집계.
(RESULT_backfill / RESULT_backfill_dongjak)

### (8) ★feature importance — 균형 다축
공시지가 28% · **노후도 27%** · 호수밀도 22% · 역세권 13% · 기타 10%. **노후도 단일 모델 아님**.
national swap이 건드리는 건 노후도+호수밀도 ≈ 49%, 나머지 51%(공시지가·역세권·접도율)는 무관. (RESULT_product_impact)

### (9) ★제품 출력 — 닮은동네는 robust, 환경점수만 흔들림
- 닮은동네(`/report` 히어로): 동작 6클러스터 평균 교집합 **4.5/5 · 1위 3/4** = national robust.
- 환경점수(보조줄·히트맵·스크리너): national 30% 손실, backfill 25% 회복. (RESULT_product_impact)

### (10) DB
파일 + 사전계산(parquet) + 메모리 pickle 구조. Supabase는 회원 전용. **swap이 DB 재설계 불트리거**. (RESULT_db_status)

### ★종합 결론
**national swap + 닮은동네 히어로 + 환경점수 정직표기 = 상업 출시 성립.**
값 정합(노후도 99.9%)·라이선스(1유형)·핵심 출력(닮은동네) robust. 총괄표제부 추가 backfill은 **스크리너를
핵심 기능화할 때만**(현 위계는 닮은동네 히어로라 불필요).

## 3. 본 파이프라인 적용 계획 (#3)
- **#3-b-1 건물 소스 교체**: 서울 SHP → national 일반+집합 + 어댑터(컬럼 리네임) + EPSG:5186 reproject.
  `build_inference_scores`/`build_serve_context` 사전계산 재실행.
- **#3-b-2 backfill 통합**: 동작 + 나머지 구 표제부 합류. 정직표기 = **닮은동네 유지 / 히트맵·스크리너의
  backfill-빈 필지는 '데이터 부족'** 표기(거짓 낮은 점수 금지).
- **#3-c B1+ national 피처 재학습·재검증**: national 피처 분포로 모델 재적합(train-serve skew 해소) + 공간CV 재검증.
- 나머지 구 backfill(~6만 PNU)은 **블로커 아님** — 닮은동네 robust, 차오르는 대로 히트맵 품질 점증.

## 4. 남은 리스크 / 주의
- **표제부 커버리지**: 동작 52%만 확인(타 구 미측정). 번지 단위 조회 미스(대표지번·총괄표제부) 가능 → 총괄표제부
  보완 시 상한↑(현재 보류).
- **이웃집계 회복 저하**: 2-hop 집계라 빈 필지가 군집 점수를 계속 끌어내림 → backfill 효율 한계.
- **정직표기 UI**: 잔여 빈 필지를 히트맵·스크리너에서 '데이터 부족'으로 노출하는 프론트 작업 필요.
- **키 보안**: 대화서 노출된 건축HUB 키 **재발급 필요**(노출분 폐기).

## 결론
4→1유형 전환은 **기술·라이선스·제품 모두 성립**. 본코드엔 #1(빈폴리곤)·#2(호수밀도 게이트) 반영 완료.
다음은 #3(건물 소스 교체 + backfill 통합 + 재학습) — 닮은동네가 robust해 단계적 출시 가능, 총괄표제부 추가
backfill은 스크리너 핵심화 시점으로 미룸.
