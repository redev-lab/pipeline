# 호수밀도 metric이 옳은가 — 정의·basis·분리력 검증 (2026-06-23)

> _experiments/gis_swap/ 전용, 본 파이프라인·git 미변경. "임계 재보정(서울 옛 통과 재현)=노이즈 보존"이라 거부하고,
> metric 자체의 타당성부터 검증.

## (a) 법적 정의 vs 현재 metric — ★불일치
- 법적 원천(도정법 시행령 별표1 / 국가법령정보센터): **호수밀도 = 정비구역 1ha당 건축물 "동수"**. 단 산정 규칙:
  ① **공동주택·다가구는 "세대수가 가장 많은 층의 1세대를 1동"으로 환산**(즉 다세대 빌라 1동 = 여러 동으로 계상).
  ② **신발생 무허가건축물은 동수에서 제외.**
- 현재 우리 metric(stage1.py:49 `len(cbld)/area_ha`) = **건물 폴리곤 raw 카운트**. 코드도 자인(stage1.py:23 "동 단위 —
  호 아님, 세대 미반영"). → **불일치 2중**: 다세대·공동주택을 **과소**(1폴리곤=1동, 법은 세대층 환산) + 무허가·부속
  빈 폴리곤을 **과대**(법은 제외).

## (b) basis 정합(빈 폴리곤 제거) 후 소스 격차 — 안 좁혀짐, 둘 다 붕괴
실건물(연면적>0)만으로 지정구역 호수밀도 재계산:

| | 지정 밀도(중앙) | 지정 통과율(≥60) | national/서울 비 |
|---|---|---|---|
| 서울 raw(전체 폴리곤) | (이전) 통과 23% | 23% | 1.00 |
| **서울 실건물** | **8 동/ha** | **9%** | 1.00 |
| **national 실건물** | **4 동/ha** | **4%** | **~0.5** |

- 빈 폴리곤 빼니 **서울 통과율 23%→9%**(raw 통과의 대부분이 빈 폴리곤이 만든 허상). national/서울 격차는 raw(0.51)에서
  **안 좁혀짐**(실건물도 ~0.5) — basis 정합만으로 소스 일치 불가, 둘 다 통과율이 바닥으로 붕괴.

## (c) ★지정/비지정 분리력 — 실건물 동수밀도는 식별 실패(역방향)
지정 1,093구역 vs 동일면적(2.83ha) 비지정 원 1,093개의 호수밀도 분리 AUC:
- 서울 실건물: 지정 중앙 **8** vs 비지정 33 → **AUC 0.325**
- national 실건물: 지정 중앙 **4** vs 비지정 27 → **AUC 0.256**
- **둘 다 AUC < 0.5 = 지정구역이 비지정보다 실건물 동수밀도가 오히려 낮다(역방향).** 재개발 지정지 = 노후 저밀
  단독·빌라(폴리곤 듬성), 비지정 built-up = 신축·고밀(폴리곤 빽빽). → **분포에서 threshold가 자연히 안 나온다.**
- (caveat) 비지정 표본을 건물점 주변에서 뽑아 밀집 쪽으로 편향 → AUC 절대값은 과장 가능. 단 **지정 통과율 9%·지정
  중앙 8(≪60)** 는 표본과 무관하게 robust: 실건물 기준 지정구역은 60동/ha를 거의 못 넘는다.

## (d) ★권고 — 임계 재보정 X, basis 정합 X, **metric 교체**
- **임계 재보정**(서울 옛 통과 재현) = 빈 폴리곤 노이즈를 정답으로 굳히는 것 → 기각(사용자 지적 옳음).
- **basis 정합**(실건물만) = 소스 격차 안 좁혀지고 분리력도 없음(역방향) → 단독으로 불충분.
- → **현 폴리곤-동수 metric을 법적 정의로 교체해야 한다**: 공동주택·다가구 **세대수 반영 동수**(다세대 빌라의 세대를
  동으로 환산) + 무허가 제외. 이게 재개발 빌라 밀집을 제대로 잡는 방향(우리 raw가 못 잡던 신호).
- **단 세대수 데이터 필요** — national 집합(D164)엔 명확한 세대수 컬럼 없음(이전 #2). 세대수 소스(건축물대장 표제부
  세대수 등) 확보 전엔 **호수밀도를 stage1 하드 요건에서 빼고 노후도+ML 모델에 위임**하는 것이 정직(현 metric은
  분리력 없는 노이즈라 빼도 손실 적음, 오히려 거짓 통과 제거).

## 1유형 swap 관점 종합
- 노후도: swap 안전(99.9% 동일).
- 호수밀도: **swap 이전에 metric 자체가 문제**(법 정의 불일치 + 분리력 없음). national이 더 나쁜 게 아니라 둘 다
  부적합. → swap 차단 요인이 아니라, **호수밀도 metric을 세대-기반으로 교체하거나 하드 요건서 제외**가 선결.
