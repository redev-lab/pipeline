# 재개발 투자 판단 시스템 (redev)

임의의 주택 주소를 입력하면 그 일대가 **(1) 재개발될 환경인가 (2) 되면 얼마나 오를
입지인가 (3) 언제쯤 (4) 무산·정체 리스크는 (5) 진입 가능한가**를 후보 경계·근거와 함께
지도로 제시하는 **투자 판단 도구**. 상세 설계 원칙은 [CLAUDE.md](CLAUDE.md).

> ⚠️ 투자 권유가 아니라 데이터 기반 참고치 + 근거 제공. 가치·수익 출력은 항상 추정·범위·
> 불확실성을 명시한다([CLAUDE.md §9](CLAUDE.md)).

## 두 개의 AI 심장

- **심장 1 — 구역 분할(GNN)**: 필지 인접 그래프 위 노드 분류로 "재개발 환경" 후보 경계 학습.
- **심장 2 — AVM(Stage 3)**: 실거래가로 입지가치·상승여력 학습. *(Phase 5)*

## 현황 (v1, 서울 4구: 성북·동작·은평·구로)

**Phase 0~8 완료** — 데이터 파이프라인, 심장1(구역 분할), Stage1 요건 룰셋, 심장2(입지가치),
LLM 종합 리포트(환각 0), 데모(지도·리포트·스크리너, 3-레포). v1.1 피처 재경기·v1.2 PU 포함.

### ★Phase 3 핵심 결과 — R9 통제 비교 (심장1)

`(필지, 시점)` 시점 라벨 + per-t 이웃집계 + **leave-one-district-out 공간 CV**(R3)로
대조군 사다리를 채점:

| 모델 | PR-AUC | 격전지 recall (aging=0 양성) |
|---|---|---|
| B-2 (aging 임계, 동어반복 바닥선) | 0.640 | 0.000 |
| B1 (XGBoost + 1홉 이웃집계) | 0.932 | 0.631 |
| B1+ (XGBoost + 2홉 이웃집계) | 0.934 | 0.622 |
| GNN (GraphSAGE 2층) | 0.929 | 0.606 |
| GNN + 자기지도 사전학습 | 0.928 | 0.616 |

**판정**: GNN은 베이스라인 천장(B1+ 0.934)을 **넘지 못하고 미세하게 뒤진다.** zone-block
bootstrap(양성 구역 + 음성 공간블록 모두 resample, n=1000): Δ(B1+ − GNN+pretrain) PR-AUC
중앙 **+0.006**, 95% CI **[+0.002, +0.012]** — 0 불포함, **통계적으로 유의하나 실질 효과크기는
무시 가능**. "능가" 증거는 전무하고, 구로·은평 fold 열세·성북·동작 동률. 자기지도 사전학습
리프트도 ≈0(PR-AUC **−0.001**) — **v1 피처 5차원에선 사전학습 무용**을 측정으로 확정.

이는 *모델 실패가 아니라 **데이터 병목***(R9): 유효표본 42구역 + 피처 5차원으로는 학습된
메시지 패싱이 1홉 손집계 위에 값을 더하지 못한다. **구조보다 도달범위, 도달범위보다 데이터.**

> **한 줄**: 구조보다 도달범위, 도달범위보다 데이터.

### v1 추론 모델 = **B1 (XGBoost + 1홉 이웃집계)**

가장 단순하면서 격전지 recall 최고(0.631). 추론·경계 산출은 B1로 간다.

### GNN은 보존 — v2 재도전 조건

GNN 코드(`redev/models/gnn/`)는 **삭제하지 않고 보존**한다. 아래 조건이 충족되면 재평가:

1. **전역 라벨 확장 (R7-①)**: 서울 전역으로 positive 구역 확대 → 유효표본 42 → 수백.
2. **피처 보강**: 용도지역·공시지가·역세권 거리 등 데이터대기 피처 추가(5차원 → 확장).
3. 위 둘 후 같은 LODO/지표로 B1 대비 재평가 — 그때 구조의 값이 드러나는지 측정.

(진짜 사전학습·GNN의 무대는 v1이 아니라 v2다. v1은 그 사실을 *측정으로* 확정했다.)

### ★Phase 5 핵심 결과 — 심장2 AVM(입지가치)도 같은 데이터 병목

실거래가(연립다세대 대지지분 평당가, R6 반경집계)로 입지가치를 학습. 시점분리 검증
(과거 학습→이후 거래 예측, 미래누수 0):

| | MAE(만원/평) | MAPE |
|---|---|---|
| AVM (XGBoost, v1 6피처) | 1,707 | 42.4% |
| 구평균 베이스라인 | **1,565** | **37.5%** |

**판정**: v1 AVM은 **구평균 베이스라인도 못 이긴다.** 피처 중요도 1·2위가 n_trades·agg_level
(데이터 밀도 프록시) — **v1 피처(노후도·접도·형상)는 "재개발 환경" 신호이지 "가치" 신호가
아니다.** 가치 드라이버(역세권 거리·용도지역·강남접근·학군)는 v1.1 대기.

→ **v1 입지가치 = 반경집계(`build_target`) + agg_level 신뢰도 직접**(99.3% 커버). 학습층
(`fit_avm`)은 결측 보간·SHAP 보조로 강등. 상승여력은 단위 불일치(대지지분평 vs 전용평)로
**v1 보류 → "시세 맥락"(두 평당가 병렬, 빼기 금지)**; 수식은 용적률·비례율·조합원분양가가
와야 정직 계산(v1.1).

### ★두 심장, 같은 교훈

심장1 GNN도, 심장2 AVM도 **v1 데이터에선 학습층이 단순 집계/베이스라인을 못 넘었다.**
**병목은 모델이 아니라 피처·라벨이다(R9).** v2(전역 라벨 확장 + 가치/입지 피처 보강)가 두
심장 모두의 진짜 무대 — v1은 그 사실을 강한 베이스라인을 세워 *측정으로* 확정했다.

### ★v1.1 재경기 — "피처냐 라벨이냐"를 해부

직교 피처 3종(공시지가 as-of-t·용도지역·역세권)을 넣고 같은 LODO·ablation으로 재측정:

| | v1 | v1.1 |
|---|---|---|
| B1+ 격전지 recall | 0.622 | **0.683** ↑↑ |
| GNN PR-AUC | 0.929 | **0.913** ↓ (구로 fold 붕괴) |
| AVM vs 구평균 | −142 MAE | **−57** (격차 절반) |
| infer 과대예측 | 81% | **81%** (불변) |

- **피처가 답인 곳**: B1+ 격전지·은평 fold·AVM은 직교 피처로 개선(피처 병목 확인).
- ★**라벨이 답인 곳**: GNN은 피처 늘리니 *악화*(유효표본 42에 피처 과적합), infer 과대예측 불변
  (지정 변별은 R18 외생·R4 PU 라벨 문제). → **GNN/infer의 레버는 라벨(전역 확장), 피처 아님.**
- **leakage_ablation**: 공시지가 도움 / **용도지역 2026 스냅샷 누수 net-negative**(§13 확인) / 역세권 marginal.

**v1.1은 병목을 고친 게 아니라 *해부*했다** — 어디가 피처 병목(AVM·격전지)이고 어디가 라벨
병목(GNN·infer)인지 강한 베이스라인+ablation으로 정밀 분리. *측정 먼저*의 완성형.

### ★Phase 6 — 데이터 병목 3호 실증 (infer 과대예측)

전 노드 추론에서 B1이 **81%를 양성으로** 과대예측(중앙 확률 0.977). 원인 = **R4 PU 갭**:
학습 음성이 신축(aging≈0)뿐이라 B1이 배운 건 *"낡았나"* → 4구 대부분이 노후라 거의 다 양성.
Phase 3 caveat("쉬운 음성에 부풀려진 0.93")의 실현. 지정구역 경계 IoU 통제비교(R13):

| 후보 | IoU | 핵심부 포착 |
|---|---|---|
| B1 넓은(best-F1) | 0.294 | 0.390 |
| B1 타이트(상위10%) | 0.019 | 0.019 |
| B0 region-grow | 0.129 | 0.135 |

- ★**타이트가 더 나쁘다(0.019)** — 점수=노후도지 지정 아님. *가장 오래된 곳은 대부분 미지정*
  (R18 외생변수의 공간적 실체). **순위가 지정과 강하게 정렬되지 않음.**
- ★**B1 넓은 > B0** — 경계/IoU 무대에선 학습이 무학습을 이긴다(노드분류 무대와 다름).
- v1 후보경계 = **거친 필터**(코어 39% 포착), 정밀 경계 아님. 히트맵은 **백분위**로 표시
  (raw 0.97 포화는 정보 0). → 진짜 해법 = PU 학습(uncertain 인지) 또는 가치/입지 피처(v1.1).

### ★Phase 7 — LLM 종합 리포트 (숫자는 안 짓는다)

5종 판단(요건·환경점수·시세맥락·진입·유사구역)을 **한국어 리포트**로 합성. 핵심 설계:
**숫자 포맷팅을 LLM에 맡기지 않는다** — 파이프라인이 표시용 문자열(`[환경유사도점수=0.993]`)을
미리 만들고 LLM은 그대로 인용만. 생성 후 **정확 문자열 비교**(`verify_numbers`)로 환각 검증
→ 불일치 0건이 합격선(규칙4). Gemini(`gemini-2.5-flash`, 모델명 config 고정). 사회신호
(layer3)는 추론 시점 근거 인용용 — ★실존 구역에 합성 부정정보 금지, 신호 없으면 "신호 없음".

### ★v1.2 PU 학습 — 측정으로 기각 (타임박스 1세션)

infer 과대예측(81%)의 R4 PU 갭을 라벨 재가중(P1 신뢰가중·P2 spy)으로 풀려 시도.
**판정: 기각.** 과대예측은 81%→37~44%로 내렸으나 **known-positive recall이 −11~26%p 붕괴**
(3%p 가드 대참사 초과)하고 IoU도 악화. §0 함정의 실현 — uncertain은 음성이 아니라 "내일
지정될 미지"라, 피처로 지정/미지정을 못 가르는 한(R18 외생) 누르면 positive까지 동반 희생.
**결정적 확증: 진짜 레버는 라벨 재가중이 아니라 라벨 확장(v2, R7-①).** "안 된다"를 측정으로
확정한 것이 결과 — 모델 사이클 무한반복 금지.

### ★Phase 8 — 데모(첫 화면): 학습 동결 + 추론 구 확장

production B1+(2홉−용도지역)를 **4구로 학습·동결**한 뒤, 마포·강남 2구를 **재학습 없이
inductive 스코어링**(귀납 일반화 검증). 학습 안 한 구에서 분포가 붕괴(전부 0/1)하면 실패 신호 —
**수검 통과**: 마포 중앙 0.976·강남 0.964로 4구와 정합(강남 신축지대 0.02로 변별 살아있음).

**데모 3종**(e2e, 환각 0·전건 합격):

| 데모 | 구분 | 후보 | 점수 | 리포트 |
|---|---|---|---|---|
| 성북 최고점 | positive(학습구) | True | 0.993 | 환각 0 |
| 마포 최고점 | ★inductive(추론전용 구) | True | 0.994 | 환각 0 |
| 강남 최저점 | negative(비대상) | False | 0.015 | 환각 0 |

세 레짐(양성/귀납/음성)이 모두 의도대로 분리 — 학습 안 한 구에서도 후보를 식별하고(귀납
일반화), 신축지대는 저신뢰로 정직하게 떨군다. 스크리너 응답 68ms.

**3-레포 분해**(경계 = 단방향): `pipeline`(순수 export `redev.serve`) ← `backend`(FastAPI:
HTTP·juso·예측로그) ← `frontend`(React+Leaflet). LangGraph/Spring 같은 다리 없음 — 동적
지점이 둘뿐이라 plain 파이썬 `run()`이 맞다(§8, Spring=v2).

> **v1 한 줄 결론**: 강한 베이스라인 사다리로 *어디가 모델 한계가 아니라 데이터 한계인지*를
> 측정으로 확정했다. 두 심장(GNN·AVM) 모두 v1 데이터에선 단순 집계를 못 넘었고, 그 병목은
> **피처(AVM·격전지)와 라벨(GNN·infer 과대예측·외생 R18)** 로 정밀 해부됐다. **v2의 진짜
> 레버 = 전역 라벨 확장(R7-①).** v1은 그 답을 *측정으로* 가리켰다.

### 데모 실행 (3-레포)

`pipeline`은 순수 export(`redev.serve`)이고, 데모는 형제 레포 `backend`·`frontend`가 띄운다:

```bash
# 1) 백엔드 (redev-lab/backend) — pipeline을 직접 import
cd ../backend && pip install -e ../pipeline -r requirements.txt
uvicorn app:app --port 8000           # 기동 시 6구 ctx 1회 빌드(~3분), 이후 캐시 조회

# 2) 프론트 (redev-lab/frontend)
cd ../frontend && npm install && npm run dev   # vite proxy → :8000
```

GEMINI_API_KEY(리포트)·선택 JUSO_API_KEY(도로명 변환)는 backend `.env`. 키 없으면 지번 입력
그대로 동작.
