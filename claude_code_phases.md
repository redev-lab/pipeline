# Claude Code 단계별 세션 프롬프트

> **사용법**: 저장소 루트에 `CLAUDE.md`를 두고, 각 Phase를 **별도 세션**으로 진행. 아래 프롬프트를 하나씩 복사해 붙여넣는다. 한 세션에 여러 Phase 몰아넣지 않는다.
>
> 모든 Phase는 `CLAUDE.md`의 **절대 규칙(§2)·학습 프로토콜(§3)·리스크 마스터(§5)**를 따른다. 설계 노트(`docs/design/`) 먼저 → 승인 → 한 모듈씩 → `LEARNING_LOG.md` 적립.
>
> **Phase = 브랜치 + PR** (CLAUDE.md §10): `git checkout -b phase-N-<이름>` → 작업 → PR. 커밋은 Conventional Commits.
>
> 일정: 빡빡한 4주가 아니라 **정공법**으로 간다(시간 여유). 각 Phase는 독립적으로 완결되게 — 중간에 멈춰도 그 지점까지가 완성된 작품.
>
> **스코프 선언: v1 = 4개 구 (Phase 0~8, 자기지도 사전학습 포함) / v2 = 서울 전역 라벨 확장 + 전역 사전학습.** 전역 그래프 빌드는 R10 최대 병목이라 v1에 앞당기지 않는다.

## Phase 3 — 심장1: GNN + 베이스라인 + 정직한 평가

```
CLAUDE.md §5(R3·R7·R8·R9·R12·R13) 읽고 시작. 천천히 한 모듈씩.

1. models/baseline.py — region-growing + ★XGBoost+이웃집계 피처. 베이스라인
   부터(R9). GNN이 이걸 못 이기면 실패가 아니라 "데이터 병목" 발견.
2. models/gnn/pretrain.py — ★R7: 라벨 없는 비라벨 노드로 자기지도 사전학습
   (피처 복원/DGI류). 라벨 ~40~50개 대비 표준 처방. ★v1은 4개구 그래프의
   비라벨 노드로 시작(수만 노드라 충분) — 전역 사전학습은 전역 라벨 확장(R7-①)
   과 함께 v2. 전역 빌드는 R10 병목이라 v1에 앞당기지 않는다.
3. models/gnn/model.py — GraphSAGE, ★얕게(2층·작은 차원) + 강한 정규화(R7).
   inductive(학습 안 한 구 추론 가능). 사전학습 → 파인튜닝.
4. train.py — ★R8: focal/weighted loss(극단 불균형). 정확도 지표 금지.
5. infer.py — 노드 확률 → ★R12: 기본 출력은 "확률 히트맵 + 후보 클러스터".
   매끈 폴리곤은 후처리(alpha shape + 최소면적 + 도로망 스냅핑) 보너스.
6. eval/spatial_cv.py — ★R3: 구역 hold-out + 버퍼(test 경계 N홉/~500m 안
   필지 train 제외, 버퍼폭은 layer 수에 맞춤). eval/metrics.py — ★R8·R13:
   PR-AUC·F1 + IoU + 핵심부 포착률 + 사람 기준선(IoU 천장 해석).

통제 비교(GNN vs region-growing vs XGBoost) 결과를 어느 쪽이든 LEARNING_LOG에
정직하게. 이기는 게 목표 아니라 정직한 비교가 목표.
```

## 세션 운영 팁

- **막히면 멈추고 물어보게**: 프롬프트에 "확신 없으면 추측 말고 물어봐"를 넣으면 환각 설계가 준다.
- **설계 노트는 협상의 장**: `docs/design/*.md`가 올라오면 대충 승인 말고 "왜 A 안 하고 B 했어?"를 되물어라. 그 되묻기가 학습의 절반.
- **LEARNING_LOG는 면접 자료**: 매 Phase 적립분이 그대로 포트폴리오 방어 스크립트.
- **리스트를 무한히 키우지 말 것**: §5 18개는 충분하다. 🔴 라벨 정합성 + 빌라 집계가 must, 나머지는 "있으면 좋은 엄밀함". 그만 늘리고 만들어라.
