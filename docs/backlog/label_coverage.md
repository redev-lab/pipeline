# 라벨 커버리지 한계 — 측정·백로그 (2026-06-19)

> 계기: 정릉동 170-1(정릉골/정릉3동 주택재개발, 실제 지정)을 시스템이 "지정 아님"으로 판정.
> 원인은 모델이 아니라 **pos 라벨 원천(의제처리 UQ181)이 서울 전체 지정구역의 부분집합**이라는 데이터 한계(R7).
> 본 문서는 **조사·기록 전용**(구현 아님). 다음 우선순위는 사용자가 정함.

## 1. 커버리지 실태 (학습 4구: 성북·동작·은평·구로)

| 항목 | 수치 | 비고 |
|---|---|---|
| 우리 라벨(pnu_zone) 지정 zone | **46** (성북15·동작12·은평16·구로3) | 의제처리 UQ1221/1222 → 시점·필지조인·dedup 통과분 |
| UQ181 의제처리 재개발(UQ1221+UQ1222) 폴리곤 | ~73 (4구) | **→ 46만 라벨화(~63%)**. 나머지는 시점(t)·필지 교차·dedup 손실 |
| UQ181 밖(일반 주택재개발) | ★측정 불가(원천에 없음) | jeonbisaeop으로 최소 ~8개 식별, 실제 더 많음 |

### 두 층위의 누락
- **(A) 원천 내 손실 (~73→46)**: 의제처리 재개발 폴리곤조차 ~37%가 라벨에서 탈락 — 시점 미상·필지 미교차·완공/철거 오염(R2)·분합필(R11). 일부는 정당한 drop(오염 차단), 일부는 복구 여지.
- **(B) 원천 밖 누락 (근본)**: **의제처리(UQ181)에 *아예 없는* 구역** — older 일반 주택재개발이 정상 지정됐으나 '의제처리' 법적 범주가 아니라 미수록.
  - UQ181 4구 title에 **0건**으로 확인된 실제 지정구역: **정릉3동(정릉골)·장위3·장위14·돈암6·안암2·신길음1·동소문2·성북2** (jeonbisaeop 2021 교차로 식별, 더 있을 것).
  - 반면 노량진·흑석1/11 등은 UQ181에 **있어 커버됨**(초기 구역명 매칭의 false-miss였음 — 단순 이름매칭 커버리지%는 신뢰 말 것).

### 정직한 결론
우리 라벨 = **"최근 의제처리(2024-25)로 처리된 재개발구역" 슬라이스**. older 관행 지정 구역(정릉골 등)은 통째 빠짐.
→ "지정 아님"은 **"우리 데이터에 없음"**일 뿐. (출력은 2026-06-17 정직성 수정으로 이미 완화: 헤더·caveat·R7)

## 2. 백로그 — v2 전역 라벨 확장 (근본 해결)

- **이게 누락의 근본 해결**: 의제처리뿐 아니라 **전체 정비구역(일반 주택재개발·재정비촉진·공공재개발 포함)** 을 라벨로.
- **우선순위 근거 = 정릉동 사례**: 실제 지정구역을 "미확인"으로 떨구는 false-negative가 사용자 신뢰를 직접 훼손. 전역 라벨이 진짜 레버(R7-①).
- 범위: 서울 전역 + (A)원천 내 손실 복구 + (B)원천 밖 경계 입수.

## 3. 단기(v1.x) 싸게 보강 가능한가 — 조사 결과

**질문**: 의제처리 외 공개 정비구역 목록으로, t-전쟁 없이 "지정 여부 판정"만 보강 가능한가?

| 소스 | 보유? | 한계 | 판정 |
|---|---|---|---|
| **jeonbisaeop**(정비사업 데이터 2021) | ✅ 보유 | 구역명 리스트(4구 17 재개발), **필지 희소(1구역≈1행)·경계 없음** | 필지 단위 '지정 여부' 정밀 보강 **불가**. 동 단위 약한 힌트만 |
| 가로주택(UQ1812, 4구 25폴리곤) | ✅ UQ181 내 | 재개발과 사회학 다름(소규모) | REDEV_SCLAS 확대는 싸나 **부적절**(별 등급이면 몰라도) |
| 정비구역 **경계(폴리곤)** 공개 SHP | ❓ 미확인 | — | ★**관건** — 누락 older 구역의 경계가 있으면 *lookup만* 추가(GNN 재학습·t-전쟁 불필요)로 "지정 여부" 보강 가능 |

### 조사할 경계 소스(미착수)
- 서울 **열린데이터광장**: "정비(예정)구역 도형/현황" 류 SHP/GeoJSON.
- **정비사업 정보몽땅**(cleanup.seoul.go.kr) GIS export — 정비구역 경계의 정전(正典).
- 토지이음/**LURIS** 정비구역 지형도면.

### 권고
- **싼 보강의 관건 = 누락 구역 *경계* 입수**. 이름 리스트(jeonbisaeop)만으론 필지 판정 불가.
- 경계 SHP를 구하면 **v1.x에서 "지정 여부 lookup" 보강 가능**(zone_boundary에 소스 추가 = §11 소스 추상화, GNN 무관).
- 경계 입수가 어려우면 **v2 전역 라벨 확장으로 통합**.
