# 설계 노트 — Foundation (config 계층 + PNU 표준화 계층)

> 규칙1(설계 먼저)에 따라 **구현 전에** 쓰는 노트. 작업자 승인 후 구현 착수.
> 대상 범위: ① §4 디렉토리 골격, ② `config/`, ③ `data/pnu.py`.

---

## 0. 이 노트가 정하는 것 (한 문단)

이번 모듈은 파이프라인의 **바닥 두 장**을 깐다. (1) **config 계층**은 "어느 구를
돌지·어떤 임계값을 쓸지"를 코드 밖으로 빼서, 서울 전역 확장이 *코드 수정이 아니라
설정 변경*이 되게 한다(규칙7). (2) **PNU 표준화 계층**은 출처가 제각각인 데이터
(건물GIS·실거래가·정비구역경계…)를 **하나의 19자리 키**로 정규화해, 이후 모든
조인이 이 키 위에서만 일어나게 한다(규칙6). 두 계층 다 모델이 아니라 **데이터
규율**이고, 여기가 흔들리면 §5의 데이터 지뢰가 그대로 터진다.

## 1. 이 모듈이 하는 일 (①)

### 1-1. 디렉토리 골격
§4의 9개 컴포넌트 매핑을 빈 패키지(+`__init__.py`)로 생성한다. 이번 세션은 **골격만**
— `config/`와 `data/pnu.py`만 실제 내용을 채우고, 나머지(`graph/`, `models/` …)는
빈 패키지로 자리만 잡는다. 이렇게 하면 import 경로(`from redev.data.pnu import ...`)가
처음부터 고정되어, 뒤 모듈들이 "어디에 앉을지"를 두고 헷갈리지 않는다(규칙2).

### 1-2. config 계층 (`config/`)
- **`districts.yaml`** — 학습 4개 구(성북·동작·은평·구로) + 추론 추가(마포·강남).
  각 구를 `시군구코드`(5자리)와 함께 명시. 코드 어디에도 구 이름/코드를 박지 않는다.
- **`legal_thresholds.yaml`** — 노후도·접도율·호수밀도 등 법정 임계값. 지금은 기획안
  기준값으로 채우되, **각 값마다** ⓐ 조항 출처 주석 + ⓑ "구현 착수 시 현행 조례/
  시행령 재확인" TODO를 단다(규칙5).
- **`__init__.py` (config 로더)** — YAML을 읽어 파이썬으로 주는 단일 진입점.
  "설정은 여기서만 읽는다"를 강제하는 좁은 문(narrow gate).

### 1-3. PNU 표준화 계층 (`data/pnu.py`)
- 어떤 타입(str/int/float)으로 들어와도 **19자리 zero-padded 문자열**로 정규화.
- 19자리·숫자 구성·법정동코드 유효성을 **검증**(invalid는 통과시키지 않고 드러낸다).
- PNU를 의미 단위로 **분해**(법정동코드·산여부·본번·부번·시군구코드).
- 시군구코드로 **필터링**(config의 자치구 리스트와 맞물림).

## 2. 왜 이렇게 설계했나 (②) — 핵심 결정 3개

### 결정 A. PNU는 "문자열"이고, float은 받지 않는다 (방어선 2층)
PNU는 19자리(~1.1e18). `float64`의 안전 정수 한계는 2^53 ≈ 9.007e15(약 16자리)로,
**19자리 PNU는 이 한계를 한참 넘는다.** 즉 PNU가 float로 들어온 시점엔 끝자리가
*이미* 반올림돼 깨진 상태다 — 복구 불가능하다. CSV/Excel에서 19자리 숫자를 무심코
읽으면 pandas가 `float`로 추론(`1.11290e+18`)하고, 선행 0(부번 등)도 사라진다.

여기서 **흔한 함정**: `normalize_pnu`가 float을 받아 `str(int(x)).zfill(19)`로
coerce하면, 깨진 float을 **"19자리짜리 그럴듯한 가짜 PNU"**로 둔갑시킨다. 이건
크래시보다 나쁘다 — 조인이 *조용히 틀린 필지*에 붙어 §5-③ 최악 시나리오가 된다.

> 그래서 방어를 **두 층**으로 나눈다:
> 1. **1차(진짜 방어선) — ingestion에서 `dtype=str`.** 숫자가 float이 될 기회
>    자체를 차단. PNU 컬럼은 읽는 순간부터 문자열. (Phase 1 ingest의 불변식.)
> 2. **2차 — `normalize_pnu`는 float을 *거부*(raise).** coerce하지 않는다.
>    "float이 여기 왔다 = ingestion이 dtype=str을 안 썼다"는 버그 신호를 시끄럽게
>    드러낸다. `str`·`int`(파이썬 int=임의정밀도, 무손실)만 받는다.

### 결정 B. config는 "읽기 단일 진입점"을 둔다
YAML을 여기저기서 `yaml.safe_load` 하면, 자치구 리스트의 출처가 분산되어 규칙7이
무너진다. `config/__init__.py`에 `load_districts()`·`load_thresholds()` 하나씩 두고,
**다른 모든 모듈은 이 함수로만 설정을 받는다.** "서울 전역 확장 = districts.yaml 한
줄 추가"가 성립하려면, 읽는 입구가 하나여야 한다.

### 결정 C. 임계값은 값과 출처를 분리하지 않는다
`노후도: 0.6` 만 적으면 6개월 뒤 "이 0.6 어디서 왔지?"가 된다. YAML 주석에
**조항(예: 도시 및 주거환경정비법 시행령 §X, 서울시 조례 §Y)**과 **재확인 TODO**를
값 바로 위에 붙여, 값과 근거가 물리적으로 같은 줄 묶음에 살게 한다(규칙5).

## 3. 검토했지만 버린 대안 (③)

| 대안 | 왜 버렸나 |
|---|---|
| PNU를 int/bigint로 보관 | 결정 A. float 경유 손실·선행 0 소실. bigint도 CSV 왕복에서 깨질 여지. **문자열이 손실 없는 유일한 표현.** |
| config를 `.py` 상수 파일로 | 비개발자(법·데이터 검수자)가 못 고친다. YAML이 "설정은 데이터"라는 경계를 더 분명히 함. 단 로딩은 파이썬이 검증. |
| 자치구를 Enum으로 하드코딩 | 규칙7 정면 위반. 구 추가가 코드 변경이 됨. → YAML + 로더. |
| 임계값을 코드 기본값으로 두고 config는 override만 | 매직넘버가 코드에 남음(규칙5 위반). **단일 출처는 config여야** 한다. |
| pandas의 dtype 추론에 맡기고 나중에 캐스팅 | 이미 float로 추론된 뒤엔 복구 불가. 표준화 함수가 **읽는 시점에** 강제해야 함. |

### 4-2. `config/__init__.py` — 함수 분해
| 함수 | 입력 → 출력 | 역할 |
|---|---|---|
| `load_districts()` | () → dict | districts.yaml 파싱. `{학습: [...], 추론: [...]}` 형태로 반환. 자치구 정보의 단일 출처. |
| `load_thresholds()` | () → dict | legal_thresholds.yaml 파싱. 임계값의 단일 출처. |
| `training_districts()` / `inference_districts()` | () → list[dict] | 학습/추론 구 리스트 헬퍼(시군구코드 포함). 호출부 편의. |

### 4-3. `data/pnu.py` — 함수 분해
| 함수 | 입력 → 출력 | 역할 (파이프라인에서) |
|---|---|---|
| `normalize_pnu(raw)` | str\|int → str(19) | 2차 방어선. `str`·`int`만 19자리 zero-pad 문자열로. **`float`은 raise**(이미 정밀도 손실 → coerce하면 가짜 PNU 양산, 결정 A). 1차 방어선은 ingest의 `dtype=str`. |
| `is_valid_pnu(pnu)` | str → bool | 19자리·숫자·법정동코드 형식 검증. 깨진 PNU를 조용히 흘리지 않게. |
| `parse_pnu(pnu)` | str → dict | 의미 분해: `{법정동코드, 시군구코드, 산여부, 본번, 부번}`. (산여부=필지구분 5번째 자리: 1=일반,2=산) |
| `sigungu_code(pnu)` | str → str(5) | 시군구코드만 추출. config 자치구 필터와 직접 맞물림. |
| `filter_by_districts(pnus, codes)` | (iterable, set) → list | 자치구 설정으로 필지 거르기. "4개 구만"이 코드가 아니라 설정에서 옴. |

> **PNU 구조 메모(구현 시 주석으로):** 19자리 = 법정동코드10 + 필지구분1 + 본번4 +
> 부번4. 법정동코드10 = 시도2 + 시군구3 + 읍면동3 + 리2. 시군구코드(5) = 시도2+시군구3.
> 서울=11. (성북 11290 / 동작 11590 / 은평 11380 / 구로 11530 / 마포 11440 / 강남 11680)
