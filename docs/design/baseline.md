# 설계 노트 — `models/baseline.py` (Phase 3, 심장1 대조군)

> 규칙1: 코드 전에 이 노트. 승인 후 구현. 충돌 시 CLAUDE.md 우선.

## 1. 이 모듈이 하는 일

심장1(GNN 노드 분류)의 **정직한 대조군**을 만든다. 같은 입력(필지 그래프 + v1 시점
피처), 같은 라벨, 같은 공간 CV 위에서 **GNN이 아닌** 두 모델을 돌려 "GNN의 메시지
패싱이 실제로 값을 더하는가"를 측정한다.

두 대조군:
- **B0 — region-growing** (규칙 기반, 무학습): "낡은 필지에서 출발해 인접한 낡은
  필지로 번져나가는" 결정론적 영역 확장. *모델이 아무것도 학습하지 않아도 공간
  인접성만으로 어디까지 가나*의 바닥선(floor).
- **B1 — XGBoost + 1홉 이웃집계** (지도학습, 비-그래프): 노드별로 자기 피처 +
  **그래프 이웃의 피처 집계(mean/max)**를 손피처로 만들어 부스팅에 먹인다.
  *이웃 정보를 "수동으로 한 홉" 넣은* 강한 대조군.
- **B1+ — XGBoost + 2홉 이웃집계** (ablation): GNN이 2층=수용영역 2홉이라, B1이
  1홉만 보면 "구조 우월"과 "한 홉 더 봄"이 교란된다. B1+는 집계를 한 번 더 적용
  (집계의 집계)해 GNN과 **도달범위(홉수)를 맞춘** 대조군. 메인 비교 아니라 ablation
  한 줄. → 3점 비교로 원인 분리:
  - `GNN > B1+ > B1` → 학습된 가중 메시지패싱(구조)이 진짜 값을 더함
  - `GNN ≈ B1+ > B1` → 값은 도달범위였지 구조가 아님 — 이것도 정직한 발견(R9 부합)
- **B-2 — aging 임계 단독** (무학습 동어반복 바닥선): `aging > 컷 → 양성`. 학습 없음.
  ★수검(2026-06-11)에서 음성 24,338 중 24,314가 신축파생(aging로 라벨링됨)이라, 이
  바닥선이 곧 **"라벨링 규칙 되외우기"의 정확한 점수**다. B0/B1/B1+/GNN이 B-2 위로
  얹는 만큼이 **'동어반복 너머의 값'**. 비용 5줄. 컷은 config(규칙5).

## 2. 왜 이렇게 설계했나 (R9 — 정직한 비교)

GNN의 유일한 구조적 무기는 **메시지 패싱**(이웃의 상태를 노드가 흡수). 만약 B1에게
노드 *자기* 피처만 주면 GNN은 "아키텍처가 우월해서"가 아니라 단지 "이웃을 봐서"
이긴다 — 비교가 거짓이 된다. 그래서 **B1에게도 이웃 피처를 집계해 같은 정보를 쥐여준다.**
그러면 비교가 묻는 진짜 질문은:

> "학습된 메시지 패싱(여러 홉·가중)이, **한 홉 수동 이웃집계 + 부스팅**보다 나은가?"

이게 R9의 정수다. **GNN이 B1을 못 이기면** → v1에선 "이웃집계 손피처면 충분, GNN의
다홉 구조는 이 데이터에선 과잉" = **데이터가 병목이라는 발견**으로 정직하게 방어.
(이기는 게 목표가 아니라 정직한 비교가 목표 — §7.)

### 검토했지만 버린 대안
- **B1에 자기 피처만 (이웃집계 생략)**: GNN이 자명하게 이김 → 비교 무의미. 버림.
- **B1을 로지스틱 회귀로**: 부스팅이 비선형·상호작용을 잡아 더 강한 대조군. R9는
  "약한 베이스라인"이 아니라 "센 베이스라인"을 요구. XGBoost 채택.
- **region-growing을 떼고 XGBoost만**: B0는 "무학습 공간 확장"의 바닥선이라 GNN의
  IoU를 해석할 좌표축을 준다(R13 사람기준선과 같은 역할). 둘 다 둔다.

## 3. 라벨 사용 (R4 — PU 구분)

라벨 테이블 certainty 3종 중 **학습에 쓰는 것**:
- `positive` (의제처리, label=1) → **양성**
- `reliable_negative` (해제·신축파생, label=0) → **음성**
- `uncertain` (노후미지정) → ★**학습에서 제외**. 음성으로 쓰면 R4 위반(미지정은
  "재개발 안 됨"이 아니라 "아직 모름"). v1은 positive vs reliable_negative 이진분류.
  uncertain은 추론 때 점수만 매긴다(PU 정식 처리는 v1.1).

## 4. 완공 9구역 drop (학습 전 확정 결정)

`contaminated=True` 130필지(9구역) = 철거→신축 완료라 as-of-t 피처가 NaN(추정 채움
금지). **데이터 로더에서 `contaminated=True` 필터로 제외** — 라벨 테이블 행은 유지
(폐기 아님), v1.1 말소대장 부활 시 필터만 풀면 복원. §13 생존편향 기록 유지.
→ 구현 후 **최종 학습 행수를 수검에 보고**.

## 5. 파일 분해 (어떤 함수가 무슨 역할)

```
models/baseline.py
├── load_training_matrix(...)   # 조립+캐시: (label ∩ graph node) ⋈ features
│                               #   - contaminated drop, uncertain drop
│                               #   - 결과를 _data/processed/train_matrix.parquet 캐시
│                               #   - 반환: X[pnu,t,피처], y, graph(edge_index), pnu_to_idx
├── aggregate_once(feat, neighbor_lists)   # ★R9 핵심(순수함수). 단일-t 피처에
│                               #   이웃 mean·max 부착. 테스트 쉬움(수검③ 손계산)
├── build_neighbor_features(tm, parcels, buildings, *, hops)  # ★(A) per-t 시점정합
│                               #   t별로 묶어 이웃피처를 그 t 스냅샷에서 흡수(R1 누수↓)
│                               #   B1=1홉, B1+=2홉(집계의 집계). 결과 캐시
├── aging_floor(X, *, cut)      # ★B-2: 무학습 동어반복 바닥선. aging>cut→양성. 5줄
├── region_grow(graph, seeds, cfg)   # B0: 낡은 필지 seed → 인접 낡은 필지 확장(BFS)
│                               #   임계값(노후도 seed/확장 컷)은 config (규칙5)
├── fit_xgb(X, y, *, split)     # B1/B1+: XGBoost. scale_pos_weight(R8 재프레이밍 §6)
│                               #   early stopping(CPU·환경요청). 좁은 탐색(3~4조합)
└── (평가는 eval/spatial_cv.py가 split을 주입 — baseline은 split-agnostic)
```

**왜 캐시(parquet)인가**: Phase 3는 하이퍼파라미터 3~4조합 + B0/B1/GNN을 여러 번
돌린다. 매 실행마다 GIS건물통합정보 zip(수십만행)을 재적재하면 분 단위 낭비. 조립
행렬을 한 번 만들어 parquet로 굳히면 이후 실행은 초 단위. (R10 병목 회피의 연장.)

## 6. R8 — 불균형 (실측 재프레이밍, 2026-06-11)

★수검 실측: 학습셋은 **positive 15,403 : neg 24,338 = 1.6:1 (거의 균형)**. R8이 가정한
"극단 불균형"이 아니다 — uncertain(노후미지정 56,428)을 PU로 제외(R4)하니 학습셋이
균형을 이룸. **극단 불균형은 학습이 아니라 *추론*(전체 141,217 노드 스코어링)에 산다.**
→ `scale_pos_weight=n_neg/n_pos`는 두되 거의 1, focal loss는 학습엔 불요. **단 지표는
여전히 PR-AUC·F1**(배포 현실이 불균형, 정확도 금지). baseline.py는 확률만, 지표는 평가 모듈.

### 6.5 ★격전지 지표 — aging=0 positive recall (R9 진짜 결승선)

★수검: positive의 aging 중앙값=0.0 — **양성의 ~절반이 자기 건물론 안 낡았다**(빈 필지·
신축 혼재). 이들은 자기피처로 음성과 구분 불가, **오직 이웃(메시지패싱)으로만 양성**.
→ 모델별 **aging=0 positive에 대한 recall을 분리 리포트**(eval/metrics.py):
- **B-2**(aging 임계): 정의상 **0%** (aging=0은 절대 못 잡음)
- **B1/B1+**: 이웃집계로 일부 포착
- **GNN**: 여기서 얼마나 더 잡나 = *동어반복 너머, 구조의 값*의 핵심 측정

이 한 칸이 R9 비교의 승부처. (메인 PR-AUC가 비슷해도 이 칸이 갈리면 결론이 갈린다.)

## 7. R3 — 공간 CV 의존성 (경계 명확화)

baseline.py는 **split을 만들지 않는다**. train/test 인덱스를 인자로 받아 그 위에서만
fit/predict. 구역 hold-out + 버퍼(R3)는 `eval/spatial_cv.py`가 책임 — baseline은
그게 주는 split을 그대로 쓴다. (관심사 분리: 모델 ≠ 검증설계.)

## 8. 환경 제약 반영 (CPU 노트북)

- XGBoost `tree_method="hist"`, `n_jobs=-1`, CPU. (GPU 가정 코드 없음.)
- **early stopping 필수** (`early_stopping_rounds`), eval_set로 검증손실 모니터.
- 하이퍼파라미터 **좁은 수동 탐색**: max_depth {3,5} × learning_rate {0.05,0.1} 정도
  4조합 수동 비교(그리드서치 자동화 금지 — 학습 목적·시간).
- **학습 1회 wall-clock을 LEARNING_LOG에 기록** (v2 GPU 필요성 판단 근거).

## 9. 수검 게이트 (규칙9 — 구현 후 통과 항목)

1. **학습 행수 보고**: reconcile→contaminated drop→uncertain drop 후 최종 X 행수 +
   positive/negative 개수 + 불균형비. (네 요청 항목.)
2. **R9 동일피처 증명**: B1이 쓰는 피처 집합 == GNN이 쓸 피처 집합(이웃집계 포함)을
   코드로 대조 출력. 피처 누수(라벨 파생 피처) 0 확인.
3. **이웃집계 정합**: 한 노드를 골라 손으로 이웃 평균을 계산 → neighbor_aggregate
   출력과 일치. (블랙박스 금지 §3.)
4. **B0 sanity**: region-growing이 낡은 구역에서 실제로 번지는지(고립 필지만 잡지
   않는지) 한 구역 시각/수치 확인.
5. **캐시 일관성**: parquet 재로드 == 즉시조립 결과 동일.

## 10. 흘려보낸 것 (v1.1+)
- uncertain의 PU 정식 처리(R4) — v1은 단순 제외.
- 이웃집계는 1홉(B1)·2홉(B1+)까지만. 3홉+는 안 만든다 — GNN이 2층(2홉)이라
  도달범위를 맞추는 건 2홉이면 충분. (흉내낼 수 있는 걸 흉내내고도 남는 격차가
  '구조만의 값' — 1·2홉을 일부러 흉내내 그 격차를 드러낸다.)
- region-growing의 정교한 도로망 스냅·alpha shape는 infer 후처리(R12)로 미룸.
