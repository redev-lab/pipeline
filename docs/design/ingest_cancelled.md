# 설계 노트 — `data/ingest/cancelled.py` (해제구역 → reliable negative)

> 규칙1. 해제구역의 `위치(지번)`을 PNU로 해석해 **reliable_negative** 라벨을
> 만든다. 상위 설계: [labels.md](labels.md) §4-2. parcels의 jibun index를 쓴다.

## 1. 하는 일 (①)
해제 CSV(176건, 4구 38건)의 `위치(지번)` 문자열을 견고하게 파싱해 parcels의
`(시군구,동,본번,부번)→PNU` 인덱스로 PNU를 찾고, `t=해제일자`인 reliable_negative
행을 만든다. naive 파서로는 매칭 50%(19/38)였고, 이 모듈의 목표는 그걸 끌어올리는 것.

## 2. 왜 견고한 파서가 필요한가 (②) — 50% 미스의 실제 원인
앞서 parcels 검증에서 잡힌 미스 패턴(전부 파서 문제, 인덱스는 정상):
| 패턴 | 예시 | 처리 |
|---|---|---|
| 공백 없음 | `안암동2가59`, `정릉동170-1` | 한글(동/가)과 숫자 경계로 분리 |
| 일대/일원 접미 | `장위동 233-42일대`, `증산동 205-33 일대` | strip |
| 산(임야) | `상도동 산65` | 산 표기 인식 → 필지구분 2로 조회 |
| ★행정동≠법정동 | `상도2동`(행정동) → `상도동`(법정동) | 동 앞 숫자 strip(행정동 패턴) |

핵심 함정은 **행정동/법정동 불일치**: 지적도(parcels)는 *법정동*인데 해제 위치는
*행정동*("상도2동")일 수 있다. → "동 앞에 숫자가 붙으면 행정동"으로 보고 숫자 제거
(`상도2동`→`상도동`, `정릉3동`→`정릉동`). 단 `삼선동1가`처럼 *동 뒤* 숫자/가는
법정동이므로 보존(숫자 위치로 구분).

## 3. 설계 결정 (②)
- **파싱 규칙은 함수로 분리**: `_parse_location(위치)` → `(법정동, 본번, 부번, is_산)`.
  규칙(공백없음·일대·산·행정동)을 한 곳에서 정의해 테스트·확장 용이.
- **자치구 가드**: 해제 CSV의 `자치구` + 파싱한 법정동을 함께 키로 → 동명 중복 차단
  (parcels jibun index가 이미 (시군구,동,본번,부번) 복합키라 자연 적용).
- **WHERE = seed(대표지번) only** — 해제 CSV는 폴리곤이 없어 구역당 1필지(대표지번)만
  PNU로 해석된다(full-zone 미커버, labels.md §4.6 수용). k-hop 확장은 v1.1.
- **매칭 실패는 drop + 사유 기록**(파싱실패/지번미존재) — 추정 금지(규칙: t·PNU 안 지어냄).
- **t = 해제일자 연도**, t_source=`cancel_date`, certainty=reliable_negative, reason=cancelled.

## 4. 함수 분해 (④)
`data/ingest/cancelled.py`
| 함수 | 역할 |
|---|---|
| `load_cancelled(csv_path, jibun_index, sigungu_codes) -> (df, report)` | 메인. 해제 CSV → reliable_neg 행 + 매칭 리포트. |
| `_parse_location(loc) -> (dong, bonbun, bubun, is_san)\|None` | 위치 파서(공백없음·일대·산·행정동). |
| `_admin_to_legal_dong(dong) -> str` | 행정동→법정동 근사(동 앞 숫자 strip). |

> `_parse_location`은 향후 다른 지번 텍스트(공공재개발 위치_지번 등)도 재사용 가능 →
> 필요 시 공용 헬퍼로 승격(zone_matching과 같은 철학).

## 5. 검토했지만 버린 대안 (③)
| 대안 | 기각 |
|---|---|
| 지번에서 PNU 산식 재구성 | 산/일반 필지구분·부번 표기 깨짐 → parcels 실값 인덱스 조회. |
| 행정동→법정동 완전 매핑표 | v1엔 표 없음. 동앞숫자 strip 휴리스틱 + 미스 리포트로 충분. 표 입수 시 교체. |
| 매칭 실패를 근방 필지로 추정 채움 | 추정 금지 원칙 위반. drop+기록. |
| 해제 구역 전체를 negative로 확장 | 폴리곤 없음 → seed only(§4.6). 확장은 v1.1 옵션. |

## 6. 검증
실데이터로: ① 해제 4구 38건 파싱 성공률(목표 50%→대폭↑) ② 매칭된 PNU 샘플 육안
③ 미스 사유 분류(파싱실패 vs 지번미존재) ④ 행정동 케이스(상도2동) 회수 확인.

## 7. 미해결 / 주의
- 행정동→법정동 휴리스틱이 틀리는 동(예: 숫자가 법정동 일부) 발생 가능 → 미스
  리포트로 추적, 빈발 시 매핑표 도입.
- 해제 CSV 위치가 구역명만이고 지번이 없으면 매칭 불가 → 해당 건 drop+기록.
- seed-only라 해제 negative는 구역당 ~1필지(R4 균형은 신축파생이 떠받침, §4.6).
