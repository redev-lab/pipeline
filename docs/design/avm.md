# 설계 노트 — `models/avm.py` (+ `data/ingest/transactions.py`) — 심장2 Stage3

> 규칙1: 코드 전에 이 노트. 승인 후 구현. CLAUDE.md §5(R6·R14·R15·R16·R17)·§6·§9 우선.

## 0. 한 문단 요약

심장2의 입지가치 모델: 실거래가로 **"이 필지 일대의 대지지분 평당 단가"**를 학습하고,
**상승여력 = 비교신축 시세 − (매입가 + 추정 분담금)**을 *범위·시나리오*로만 제시한다(단정
금지, R14). 빌라·다세대가 재개발 물건 대부분이라 ★개별 호수 매핑을 버리고(R6) **필지+반경
집계**를 타깃으로 한다. 예측을 강화할수록 면책 프레이밍을 강화한다(R15 규제 역설).

## 1. 두 거래 흐름 (역할 분리)

- **연립다세대 매매**(API 15126467, `VILLA_TRADE_API_KEY`) → ★**대지지분 평당 단가**(입지가치
  학습 타깃, R6). 빌라는 대지권면적 제공 → 평당 대지지분가가 호수·층·전용면적 편차를 씻어낸다.
- **아파트 매매**(API 15126469, `APT_TRADE_API_KEY`) → **전용 평당가**(★비교신축 시세 reference,
  R17 상승여력의 윗변). 아파트는 대지권 미미 → 전용 평당가가 신축 시세의 기준.

## 2. ★R6 — 빌라 매핑 지옥 회피 (이 모듈의 심장)

개별 호수 실거래가를 PNU 1:1 매핑하면 결측 폭발 + 층·향·전용 편차 노이즈. 대신:
- **타깃 = 필지 중심 반경 `radius_m`(config, 기본 50m) 내, 최근 `window_years`(기본 3년)
  연립다세대 거래의 대지지분 평당단가 집계(median).**
- 대지지분 평당단가 = 거래금액 / (대지권면적㎡ / 3.3058) [만원/평]. ★층·전용 무관 — 땅값.
- median(평균 아님): 소수 고가·저가 거래에 강건.
- ★**계층 폴백(거짓 0 ↔ 결측천지 둘 다 회피)**: 50m에 거래 0건이면 **100m → 동 단위**로
  넓혀 값을 얻되, **어느 단계에서 채웠는지 `agg_level` 컬럼**('r50'/'r100'/'dong'/missing)으로
  기록 → 피처 겸 신뢰도 표시(넓힐수록 국지성↓). 끝까지 0건이면 타깃 결측(R16). 단계 반경은 config.

## 3. 데이터 의존 — `transactions.py` ingest 계약 (먼저 구현)

API(국토부 실거래가) → 정제 거래 테이블. 소스 추상화(§11 철학) — 출력 스키마 고정:

```text
거래 테이블 스키마:
  pnu(19) · trade_type('villa'|'apt') · deal_amount(만원) · area_m2(전용)
  · land_share_m2(대지권, villa만) · build_year · deal_ym(YYYYMM) · sigungu
출력 = (정제 거래 df, 위생 리포트: API건수·PNU매핑실패율·대지권결측율)
```

- **PNU 매핑**: API의 법정동+지번(본번-부번) → `build_jibun_index`(parcels)로 PNU(cancelled.py와
  동일 방식 재사용). 매핑 실패(지번결측·단독다가구 일부) → drop + **실패율 기록**(R6 수검).
- **범위**: 4구 + 비교신축용 인접 여유. 최근 3년+(window보다 길게 받아 시점분리 검증 여지).
- **키**: `.env`에서 `os.getenv("VILLA_TRADE_API_KEY"/"APT_TRADE_API_KEY")`. ★키는 Encoding
  형태(%포함)라 **raw로 전달**(urlencode 재인코딩 금지 — 401). .gitignore 확인됨.
- ★**API 명세(2026-06-12 실호출 검증, 가정 아님)**: 엔드포인트 `apis.data.go.kr/1613000/
  RTMSDataSvcRHTrade/getRTMSDataSvcRHTrade`(villa)·`...AptTrade/getRTMSDataSvcAptTrade`(apt).
  params: serviceKey·LAWD_CD(시군구5자리)·DEAL_YMD(YYYYMM)·numOfRows·pageNo. 필드: `dealAmount`
  (만원,콤마)·`excluUseAr`(전용)·**`landAr`(대지권, villa만)**·`buildYear`·`dealYear/Month/Day`·
  `jibun`·`umdNm`(법정동)·`sggCd`·`cdealType`(O=계약해제→제외). 호출=(시군구,연월), 4구×36월×2API
  =288건 < 일1000한도. (kind,lawd,ym) XML 디스크 캐시(재호출 회피).

## 4. 피처 (v1 가용분 + v1.1 대기 명시)

- **v1**: 대지지분(타깃 정규화에 사용)·필지면적·형상(compactness)·노후도·접도(Phase 2 features
  재사용) + **거래 파생**(반경 내 거래빈도·최근성·표본수 — R16 결측을 *피처로* 인지).
- **v1.1 대기(데이터)**: 역세권 거리·용도지역·강남/CBD 통근·학군·POI. README/출력에 "미반영" 명시.
  (Phase 2에서 area outlier·데이터대기 피처를 v1.1로 미룬 것과 동일 정책.)

## 5. ★시점 정합 검증 + 가격 상대화 (누수 차단)

- **시점 분리**(R1 정신): 거래를 연도로 갈라 **과거 학습 → 이후 예측 검증**. 미래 거래로 과거
  가치 예측 금지. 무작위 분할은 시세 추세를 커닝.
- **가격 피처 (연도, 자치구) 백분위 상대화**(labels §9 그대로): 절대 평당가 대신 "그 연도·그 구
  분포에서 몇 분위"로 상대화 → 시대·지역 시세차가 모델을 오염시키는 걸 완화.
- 공간 검증은 AVM도 구역/구 인접 누수 주의 — v1은 시점분리 우선, 공간CV는 표본 보고 결정.

## 6. "시세 맥락" (R14·R17) — ★상승여력 단정 보류, 병렬 사실만

★측정 실증(2026-06-12): `비교신축 전용평당가 − 대지지분 평당가`를 1:1로 빼면 **단위가 달라
음수 천지**로 오도된다(예시 매입5407 vs 신축4959 → 밴드 −2070~−5856). → **'상승여력' 수치를
v1에서 보류**하고 **"시세 맥락"**으로 프레이밍(단어가 숫자 하나를 기대하게 만들기 때문).
- 출력(`market_context`) = 두 사실 **병렬**: "인근 빌라 대지지분 평당 X만원(반경·신뢰도)" /
  "인근 신축 아파트 전용 평당 Y만원". **빼기 금지, 단위 다름 명시.**
- **비교신축 시세**(R17): 아파트 중 거리≤`comp_radius`·연식≤`comp_max_age`(config 공개) 전용 평당가 median.
- **상승여력 수식(신축시세 − 매입가 − 분담금)은 v1.1**: 정직 계산엔 **용적률·비례율·조합원분양가**가
  와야 한다(config `upside_v1_1.required_inputs`). ★"안 하는" 게 아니라 "재료 대기" 포지션.
- ★R15 연동: 수치 단정을 안 하니 **면책 부담도 같이 내려간다**(§9). 진단/예언 분리(§6)의 '예언' 보류.

## 7. ★R16 선택 편향 + R15 규제 역설 (정직성)

- **R16**: 거래된 물건만 보고 배운다 → 거래 빈번 지역으로 편향. 표본수·결측을 **피처+출력 경고**로
  인지("이 추정은 반경 내 거래 N건 기반, N작으면 신뢰낮음").
- **R15**: AVM이 정밀해질수록 유사투자자문 회색지대. → **예측 강화 = 면책 강화**(한 몸). 모든 가치·
  상승여력 출력에 "추정·범위·권유 아님" 프레이밍 동봉(stage1 caveat와 같은 방식).

## 8. 모델 + 검증 — ★측정 판정: v1 학습층은 부가가치 0 (심장1과 동형)

- **부스팅 + SHAP**(가치 기여). CPU `hist`(device 가정 금지).
- **검증(2026-06-12 시점분리)**: 학습 max_ym 202412 < 검증 min_ym 202501(미래누수 0). 결과:
  **AVM MAE 1707 / MAPE 42.4% vs 구평균 베이스라인 MAE 1565 / 37.5% → AVM이 못 이김.**
  피처 중요도 1·2위가 n_trades·agg_ord(데이터 밀도 프록시) — **v1 피처는 '재개발 환경' 신호이지
  '가치' 신호가 아니다.** 가치 드라이버(역세권·용도지역·강남접근·학군)는 v1.1.
- ★**v1 입지가치 출력 = `build_target`(반경집계) + agg_level 신뢰도 직접.** `fit_avm`은 결측
  필지 보간·SHAP 보조로 **강등**, "가치피처(v1.1) 후 재평가"로 못박음.
- ★**두 심장 같은 판정**(README·§13): 심장1 GNN도, 심장2 AVM도 v1 데이터에선 학습층이 집계/
  손피처를 못 넘었다 — **병목은 모델이 아니라 피처.** 정직한 데이터 병목 발견(R9 정신).

## 10. ★수검 계획 (규칙9, 구현 후 — 설계에 미리 박음)

1. **시점 분리 증명**: 학습 거래의 최대 deal_ym < 검증 거래의 최소 deal_ym (미래 누수 0 확인).
2. **4구 평당가 분포 상식**: 구별 median 대지지분 평당가 순위가 통념과 정합하나(예: 동작·성북 vs
   구로·은평 시세 통념). 어긋나면 단위·집계 버그.
3. **R6 집계 커버리지 + ★반경별 폴백 표**: PNU 매핑 실패율 + **agg_level별 채움 비율 표**
   (r50/r100/dong/missing 각각 몇 %). 50m에서 대부분 채워지면 국지성 좋음, dong에 몰리면
   국지성 약함(신뢰도 경고). 끝까지 결측율이 높으면 radius/window 재검토(데이터 기반).
4. **베이스라인 대비**: AVM MAE < "구 평균 평당가" 베이스라인 MAE (못 이기면 정직하게 보고 — 심장1
   R9와 같은 통제정신).
5. **상승여력 범위 sanity**: 분담금 시나리오 밴드가 음수~양수 걸치는지(불확실성 정직 반영), 단정값
   안 나오는지 확인.

## 11. 검토했지만 버린 대안 / 한계

- **개별 호수 1:1 매핑**: R6 결측 폭발·노이즈 → 버림. 반경 집계.
- **평균 집계**: 소수 거래 이상치 취약 → median.
- **절대 평당가 학습**: 시대·지역 시세차 오염 → (연도,구) 백분위 상대화.
- **분담금 단일 추정**: 불투명·단정은 R14·R15 위반 → 시나리오 범위만.
- **역세권/용도지역 피처**: 데이터 대기 → v1.1(미반영 명시).
