# 설계 노트 — 정비계획 결정고시 파싱 (성능 트랙 1, v1_3-gosi)

> 규칙1: 코드 전에 이 노트. 승인 후 구현. ★특히 §2 입수 경로는 결정이 필요(첫 관문).
> 규칙9: §1 라벨 인벤토리 측정을 먼저 끝냈다(아래). 규칙4: LLM은 숫자 안 짓는다.

## 0. 한 줄
정비계획 결정고시 **본문**에서 용적률·건폐율·계획 세대수·구역면적을 **추출(예측 아님, 사실 추출)**해
zone 단위로 ZoneTable·retrieval 벡터에 합류시키고 리포트 "얼마"/"사례" 칸을 채운다. 천장 리스크 0
(미래 예측이 아니라 문서에 적힌 사실), 커버리지 최대(지정 단계는 거의 모든 구역 보유).

## 2. ★입수 경로 (첫 관문 — 결정 필요)

각 zone = (고시번호, 고시일자, 사업명)을 키로 본문(PDF/HWP/HTML)을 받는다. 후보 소스:

| 소스 | 형태 | 비고 |
|---|---|---|
| **토지이음** eum.go.kr 고시정보 | 고시 본문 텍스트/PDF | 고시번호 조회. 공공·열람 자유. 1순위 |
| **정보몽땅** cleanup.seoul.go.kr | 사업별 고시·공람 HWP/PDF 첨부 | 정비사업 단위 정리, 첨부 본문 풍부 |
| 서울 전자관보/고시공고 | 고시번호 PDF | 보조 |

**제안 순서(측정 먼저·de-risk)**:
1. **수동 표본 5구역**(§5 손대조 대상과 동일): 토지이음/정보몽땅에서 본문을 손으로 받아
   `_data/raw/gosi_body/<고시번호>.{pdf,hwp,txt}`로 둔다. → 파서+StructVerify가 *실제 본문에서*
   동작하는지 먼저 증명.
2. 표본에서 추출·검증 통과율이 충분하면 → **벌크 입수 자동화** 결정(스크래핑 ToS 검토 vs 수동 다운로드).
   소스 추상화(§11 철학): 입수기 `gosi_fetch`는 (고시번호)→(본문 텍스트)만 책임, 소스 교체 가능.

> ★승인 포인트: (a) 1순위 소스(토지이음) OK? (b) 표본 5구역 수동 입수 후 자동화 판단 순서 OK?
> (c) 스크래핑이 ToS상 곤란하면 수동 다운로드 범위(예: 51구역)로 한정?

## 3. 추출 설계 (LLM은 숫자 안 짓는다 — 규칙4)

- 입력: 본문 텍스트(PDF/HWP→text). 출력: zone별 구조화 `{용적률, 건폐율, 계획세대수, 구역면적,
  (보너스)분양가·단계일자}` + 각 값의 **출처(고시번호·근거 문장)**.
- LLM 역할: 본문에서 **추출만**. 없으면 `"미기재"`(추정·창작 금지). 표/문장 어디에 있든 원문 표기 그대로.
- 숫자 정규화는 결정론 코드(파이프라인)가: "250퍼센트"·"250%"·"250.0" → 표시문자열 고정.

## 4. ★StructVerify (검증이 핵심 — 환각 0 기법 그대로)

1. **verbatim diff**: 추출한 각 수치 토큰이 본문 원문에 **글자 그대로 존재**하는지 대조
   (llm/report.py `verify_numbers`와 동일 기법). 원문에 없는 숫자 = **불합격**(드롭+로그).
2. **상식 범위 가드**: 용적률 50~600% / 건폐율 20~80% / 세대수 ≥1 / 면적 >0. 범위 밖 = 추출 오류 플래그.
3. **출처 보존**: 값마다 `{value, raw, 고시번호, sentence}` 메타 → 리포트가 "용적률 250%(서울고시 2021-76)"로 인용.
4. 결과 등급: `verified`(diff+범위 통과) / `flagged`(범위밖) / `missing`(미기재). verified만 표시 채택.

## 5. 통합 (얼마·사례 칸)

- ZoneTable에 `far(용적률)·bcr(건폐율)·planned_units·zone_area_m2` + 출처 메타 컬럼 추가.
- retrieval `build_zone_vectors`: 물리 4피처에 **규모(계획세대수·면적)** 보강(사례 변별 강화) — 단,
  결측 zone은 기존대로 평균대치(누수 방지 표준화 후 0). 피처 추가는 ablation으로 효과 측정.
- 리포트 "얼마": 현 시세맥락(평당가)에 **계획 밀도(용적률·세대수)** 사실 추가(상승여력 단정 아님, R14 유지).

## 6. 함수/파일 분해

```text
data/ingest/gosi_body.py   # 고시번호 → 본문 텍스트(소스 추상화: 토지이음/정보몽땅/수동파일). PDF·HWP→text.
nlp/gosi_extract.py        # 본문 → 구조화 추출(LLM, 미기재 허용) + 정규화(결정론)
nlp/gosi_verify.py         # ★StructVerify: verbatim diff + 범위 가드 + 출처 보존 → 등급
data/zone_attrs.py         # zone별 추출값 병합 → ZoneTable 컬럼(verified만)
  (retrieval/case_search.py·llm/report.py·orchestration/pipeline.py 에 합류)
```

## 7. ★수검 (규칙9, 구현 후)

1. **verbatim 검증 통과율**: 추출 수치 중 원문 대조 통과 %(목표 ~100%, 불합격은 드롭).
2. **범위 가드 작동**: 의도적 범위밖 값 주입 시 flagged 되는지 + 실데이터 flagged 건수.
3. **표본 5구역 손대조**: 추출값 vs 원문 직접 확인(사람 눈) — 정확도.
4. **커버리지**: 51(또는 65)구역 중 용적률/건폐율/세대/면적 각 몇 개 채워졌나.
5. **리포트 반영**: "얼마" 칸에 실제로 "용적률 X%(고시번호)"가 출처와 함께 뜨는지 + 환각 0 유지.

## 8. 검토했지만 버린 대안

- **로컬 CSV `내용` 파싱**: 측정 결과 본문 아님(전문 요약), 목표 수치 0/65 → 불가(§1).
- **LLM이 수치 추론/보정**: 규칙4 위반·천장 리스크 발생 → 추출+verbatim 검증만.
- **관리처분/분양가 우선**: 표본 적은 함정(커버리지 낮음) → v1.4 보류, 지정단계(용적률 등) 먼저.
- **단계일자 추출로 '언제' 부활**: 별도 트랙. 본 트랙은 사실 수치(얼마)에 집중.
