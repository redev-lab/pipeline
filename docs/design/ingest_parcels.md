# 설계 노트 — `data/ingest/parcels.py` (연속지적도 → 필지 스파인)

> 규칙1. 파이프라인의 **등뼈**. labels의 PNU 해석(지뢰3)과 Phase 2 인접 그래프가
> 모두 여기서 나온다. t 문제와 무관한 unblocked 모듈.

## 1. 하는 일 (①)
연속지적도 SHP(AL_D002, 898,719 필지)를 읽어 *정제 필지 테이블*로 만든다.
모든 라벨·그래프가 올라타는 PNU·geometry의 단일 출처. 더불어 **(동명, 지번)→PNU
인덱스**를 제공해, 해제구역의 `위치(지번)`을 PNU로 해석한다(지뢰3의 해제 부분).

## 2. AL_D002 매핑 (실데이터 EDA 확정)
| 원천 | 정제 | 비고 |
|---|---|---|
| A1 | `pnu` | 19자리. `normalize_pnu` 관문 통과 |
| A3 | `dong_addr` | "서울특별시 은평구 수색동" (법정동 주소) |
| A4 | `jibun` | "179-3" (본번-부번). 해제 위치 매칭 키 |
| A7 | `sigungu` | 시군구코드(5) — 구 필터 |
| (geom) | `geometry` | 폴리곤. 원천 이미 EPSG:5186 |

## 3. 설계 결정 (②)
- **`where` 절로 4구만 읽기(R10 메모리).** pyogrio `where="A7 IN ('11290','11590',
  '11380','11530')"` → 89.9만 중 해당 구만 로드. 자치구 코드는 config에서(규칙7).
- **geometry 기본 로드(스파인).** building_gis와 달리 여기선 geometry가 본체
  (그래프·공간조인). `to_target_crs`로 5186 통일(닻 재사용 — 이미 5186이라 no-op).
- **PNU normalize 관문.** A1 → normalize_pnu(19자리·float거부). 실패 행 drop+리포트.
- **(동명,지번)→PNU 인덱스는 지적도 실값으로.** 지번에서 PNU를 *재구성*하지 않는다
  (산/일반 필지구분·부번 표기 차이로 깨짐). 대신 지적도 A4(지번)·A3(동)·A1(PNU)로
  실제 인덱스를 만들어 lookup. → 해제 "수유동 711"을 (강북구·수유동·711)로 조회.
- **법정동 파싱.** A3에서 동명을 떼되, 같은 동명이 여러 구에 있으므로 항상
  `시군구 + 동명 + 지번`을 키로(예: 수유동은 강북구).

## 4. 함수 분해 (④)
`data/ingest/parcels.py`
| 함수 | 역할 |
|---|---|
| `load_parcels(path, sigungu_codes, *, with_geometry=True) -> (gdf, report)` | 지적도 → 정제 필지 테이블(스파인). where로 구 필터 + 5186. |
| `build_jibun_index(parcels) -> dict` | `{(sigungu, dong, 본번, 부번): pnu}` 인덱스. 해제 지번매칭용. |
| `_parse_dong(addr) -> str` | A3 "서울특별시 은평구 수색동" → "수색동". |
| `_parse_jibun(s) -> (본번, 부번)` | "179-3" → (179, 3), "711" → (711, 0). |

## 5. 검토했지만 버린 대안 (③)
| 대안 | 기각 |
|---|---|
| 서울 전체 89.9만 필지 통째 로드 | 메모리·속도(R10). where로 구 필터. |
| 지번에서 PNU를 산식으로 재구성 | 산/일반 필지구분·부번 표기(산39 vs 39) 때문에 깨짐. 지적도 실값 인덱스가 안전. |
| 동명만으로 매칭 | 동명 중복(여러 구) → 오매칭. 시군구+동+지번 복합키. |
| geometry 안 읽기 | 스파인이라 그래프·공간조인에 필수. 단 with_geometry=False 옵션은 남김. |

## 6. 검증
실데이터로: ① 4구 필지 수·PNU normalize율 ② geometry 5186·유효성 ③ 해제 CSV의
위치 몇 개(수유12 "수유동 711" 등)를 build_jibun_index로 PNU 찾기 — 매칭률 확인.
(이 매칭률이 곧 labels의 해제 negative 커버리지.)

## 7. 미해결 / 주의
- 깨진 geometry(self-intersect 등)는 with_geometry 경로에서 make_valid 처리(R10).
- 지번 매칭 실패(주소 표기 불일치·통폐합)는 labels의 R11(분합필)과 함께 drop+기록.
- 신통 positive는 구역명만 있어 지번매칭이 안 됨 → 별도 경로(의제처리 폴리곤
  ∩ 또는 정보몽땅 대표지번) 필요 → labels.py 설계에서 다룸(여기 책임 아님).
