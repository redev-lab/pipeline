## 0. 한 문단 요약

진입 가능성을 **결정론 룩업**으로 답한다(학습 아님): ① **토지거래허가(토허)** 적용·갭투자
가능 여부를 **물건 유형으로 분기**, ② 정비사업 **단계→잔여기간**을 범위로. 모든 값은 config
(규칙5), **토허는 수시 변경**이라 재확인 TODO를 강하게 박는다. 진단(토허=현재 사실)과
예언(잔여기간=미래 추정)을 섞지 않는다(§6).

## 1. ★토허 = "구역 룩업"이 아니라 "전역 기본 + 물건유형 분기" (§11)

2025.10 대책으로 서울 대부분 자치구가 토허로 확대 → **특정 구역 멤버십 조회가 무의미**.
v1 로직:
- **서울 전역 기본 적용**(4구 전부 토허 대상 가정, config `toheo.default_applies: true`).
- **물건 유형 분기**: 토허 대상 = **아파트** + **단지요건 충족 연립·다세대**. 일반 빌라(단지요건
  미충족)는 토허 비대상 → 갭투자(전세 낀 매입) 가능. 단지요건 임계(세대수·동수)는 config.
- 토허 대상이면 **실거주 의무 → 갭투자 불가**.
- ★**수시 변경 재확인 TODO**: 토허 지정·해제·요건은 분기마다 바뀐다 — config 값에 "고시일자
  기준·재확인" 주석. 단정적 "갭투자 가능/불가"에 **시점 단서** 동봉.

## 2. 단계 → 잔여기간 룩업 (예언, 범위·천장)

정비사업 단계(조합설립→사업시행인가→관리처분→착공→준공)별 **예상 잔여기간을 범위로** 룩업
(config 테이블). ★단정 금지(§6 예언) — "평균 N년, 변동 큼" + 외생변수(분쟁·경기) 천장 명시.
단계 입력은 외부(정보몽땅 직수집/OA-2254 보조, §11)에서 주어진다고 가정 — eligibility는 룩업만.

## 4. 진단/예언 분리 (§6)

- **진단(현재 사실)**: 토허 적용·갭투자 가능 여부 — 법·고시 기반, 높은 확신(단 시점 단서).
- **예언(미래 추정)**: 단계 잔여기간 — 외생변수 천장, **범위·"변동 큼"으로만**.
- 출력에서 둘을 라벨로 분리(섞으면 사용자가 추정을 사실로 오해).

## 6. ★수검 (규칙9, 구현 후)

1. **알려진 토허 케이스 1~2개 수동 대조**: 예) 4구 아파트 → 토허 적용·갭투자 불가 / 일반 빌라
   (단지요건 미충족) → 비대상·갭투자 가능. 실제 현행 규정과 손대조.
2. **단계 룩업 sanity**: 관리처분 단계 잔여기간 < 조합설립 단계(후반일수록 짧음) — 순서 정합.
3. **진단/예언 분리 출력**: 토허(사실)·잔여기간(추정)이 라벨로 분리되어 찍히는지.

## 7. 검토했지만 버린 대안

- **토허 구역 멤버십 테이블**: 전역 확대로 무의미·수시변경 → 전역기본+물건분기(§11).
- **잔여기간 단일값**: 외생변수 천장 → 범위. R15·§6 위반 회피.
- **단계 자동 추론(학습)**: 데이터(정보몽땅) 직수집이 정답 — eligibility는 룩업만.
