# 설계 노트 — `retrieval/case_search.py` (Phase 7, 사례검색)

> 규칙1: 코드 전에 이 노트. 승인 후 구현. CLAUDE.md §8(과한 도구 회피)·규칙4 우선.

## 0. 한 문단 요약

후보지(클러스터)를 받아 **"○○구역과 N% 유사 + 그 구역 진행 이력"**을 낸다. 비교 대상은
의제처리 지정구역 **51개뿐** — DB·pgvector는 과한 도구다. 구역 피처 벡터를 표준화해 **코사인
유사도(numpy)**로 충분. 유사도의 근거(어느 축이 닮아 N%인지)를 함께 낸다. 숫자는 데이터에서
계산만(규칙4) — LLM 미사용.

## 1. 이 모듈이 하는 일

- **구역 벡터 사전구축**: 51개 지정구역 각각을 ★**물리 피처 4개**(노후도·면적·호수밀도·접도율)
  벡터로. stage1 `cluster_metrics` 재사용 → 같은 지표축(R9 일관).
- ★**t(지정연도)는 유사도 축 제외 — 메타로만**. 질의측(후보지)은 미지정이라 t가 없다.
  현재연도로 채우면 "최근 구역 선호"라는 *미표시 가중치*가 몰래 섞인다. → 유사도는 물리
  피처만, t는 결과카드에 "○○구역(2021 지정)"으로 표시. "동시기 유사"는 장위8/9가 물리적으로도
  닮아 t 없이도 최근접으로 나와야 정상(안 나오면 잡아야 할 신호). 선택: "최근 사례 우선"은
  유사도에 섞지 말고 **명시적 정렬 스위치**로 별도 노출.
- **쿼리**: 후보 클러스터 → 같은 피처 벡터 → 51개와 코사인 유사도 → 상위 K 유사 구역.
- **진행 이력**: 각 구역의 메타(zone_type UQ1221/1222, 지정연도 t, 완공여부) 동봉. 단계별
  상세 이력(조합설립→…)은 정보몽땅 직수집 필요 → **v1.1**(여기선 가용 메타만, 한계 명시).
- **유사도 근거**: 표준화 벡터의 per-축 기여(aᵢ·bᵢ)에서 상위 축 → "노후도·호수밀도가 닮아 82%".

## 2. ★왜 DB가 아니라 numpy인가 (과한 도구 회피, §8)

비교 대상이 **51개**다. pgvector/DB는 수천~수만 벡터의 ANN 검색 도구 — 51개엔 전수 코사인이
더 빠르고 투명하다(블랙박스 없음, §2-3). 표준화(z-score)만 정확히 하면 numpy 한 줄.
- pgvector는 **v2(서울 전역 = 수백 구역)** 에서 도입 — 그때가 ANN의 무대. (§11 config-driven 철학.)

## 3. 피처 표준화 (코사인의 전제)

축마다 스케일이 달라(면적 ㎡ vs 노후도 0~1) → **z-score 표준화**(51구역 분포 기준) 후 코사인.
표준화 통계(mean/std)는 51구역에서 적합·고정(쿼리도 같은 통계로 변환 — 일관성).

## 4. 함수/파일 분해

```text
retrieval/case_search.py
├── build_zone_vectors(zones_metrics) -> (Z[51,F], meta[51], scaler)   # 표준화 사전구축
├── _featurize(cluster_metrics, scaler) -> vec[F]                       # 쿼리 벡터(같은 통계)
├── cosine_topk(qvec, Z, *, k) -> [(zone_id, sim, top_axes)]           # 상위 K + 기여축
└── search_cases(cluster, parcels, buildings, zones, *, k) -> dict      # 공개: 유사구역+이력+근거
```

## 5. 출력 형태 (구조화 — LLM이 언어화할 입력)

```json
{"query_metrics": {...}, "matches": [
  {"zone_id": "...", "similarity": 0.82, "zone_type": "UQ1221", "designated_year": 2008,
   "completed": false, "top_similar_axes": ["노후도", "호수밀도"]}, ...]}
```

## 6. ★수검 (규칙9, 구현 후)

1. **유사도 sanity**: 장위8류 쿼리의 최근접이 장위9류(인접·동시기 구역)로 나오는지 — 도메인
   상식과 정합. (4구 실제 구역으로: 같은 동·비슷한 t 구역이 상위에 오나.)
2. **표준화 정합**: 표준화 후 각 축 평균≈0·표준편차≈1(51구역). 쿼리가 같은 통계로 변환되는지.
3. **기여축 해석**: 상위 유사 구역의 top_axes가 실제로 두 벡터에서 큰 값인지(손 대조 1건).

## 7. 검토했지만 버린 대안

- **pgvector/DB ANN**: 51개엔 과한 도구·블랙박스(§8) → 전수 numpy 코사인. v2 전역서 도입.
- **표준화 생략**: 면적이 코사인을 지배 → z-score 필수.
- **단계별 진행이력 풀수집**: 정보몽땅 크롤링 = v1.1 → v1은 가용 메타(t·type·완공)만, 한계 명시.
