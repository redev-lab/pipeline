# 설계 노트 — graph/build.py + graph/features.py (심장1 입력, Phase 2)

> 규칙1. 심장1(GNN)의 입력을 만든다. 라벨((필지,시점) 테이블)은 *무엇이 positive인가*,
> 그래프는 *필지들이 어떻게 이웃인가*, 피처는 *각 시점 t의 필지 상태*를 준다.

## §3-1 핵심 개념 — 메시지 패싱이 이 그래프에서 흡수하는 것
재개발은 필지가 아니라 **블록 단위**다. 메시지 패싱은 각 필지가 이웃의 노후·접도·밀도를
흡수해 "이 필지의 *주변 블록*이 재개발 환경인가"를 임베딩에 담는다. **엣지=경계 공유**가
"같은 블록"의 조작적 정의. (1990년대 신축 한 필지도 이웃이 낡았으면 후보 — 규칙 불가, GNN 가능.)

## 1. 이 모듈이 하는 일 (①)
- **build.py**: 필지 인접 그래프. **주거계 필지=노드**, 경계 공유=엣지 → PyG `edge_index`.
  ★비주거 지목(도로·하천·구거·철도 등)은 **노드에서 제외**(수정1, §2). 구 단위 배치 +
  공간인덱스(R10). 전역 그래프 1회 구축·캐싱(추론은 조회).
- **features.py**: 노드 피처 X를 ★각 (필지, 시점 t) 행의 **t 기준**으로 계산. 노후도·호수밀도
  등 **건물 파생 피처는 전부 사용승인일≤t** 필터(보강1, R1). 그래프 구조는 정적, t는
  *피처만* 고른다(키스톤 §2-3).

## 2. 왜 이렇게 (②)
- **R10 방어(일정 최대 병목)**: 173K 필지를 한 번에 `touches` 공간조인하면 메모리·시간
  폭발. → **구 단위 배치 + STRtree 공간인덱스**(O(n²)→근방만). 전역 그래프는 1회 구축해
  캐싱, 추론은 서브그래프 조회만(수십 초 실시간 추출 회피).
- **R1 시점 정합**: features는 라벨 행의 t를 받아 aging로 *t 시점* 노후도. 미래 정보 차단.
  → **여전히 정적 노드 분류**(동적 GNN 아님): 그래프 G는 현재 필지 그래프 하나(R11 resolve,
  labels 결정4), t는 같은 노드의 *다른 피처 스냅샷*을 고를 뿐.
- **★수정1 — 비주거 지목 노드 제외**: 모든 필지를 노드로 넣으면 **도로 필지 하나가 길게
  뻗어 수백 필지와 touches → 슈퍼허브 노드**가 된다. 메시지 패싱이 도로를 타고 *블록 경계를
  넘어* 번져 §3-1의 "엣지=같은 블록" 정의가 깨진다. → **도로·하천·구거·철도 등 비주거 지목은
  노드에서 제외**(제외 지목 목록 config). 도로는 노드가 아니라 **다리**: 도로 건너는 연결은
  버퍼 옵션이 담당.
- **엣지 정의**: 주거계 노드 간 경계 공유(`touches`)가 기본. 도로로 갈린 필지는 안 닿음 →
  얇은 도로를 잇는 작은 버퍼 옵션(블록 연결성). 버퍼 폭은 config·실측(매직넘버 금지).
  ★수검의 **degree 분포(제외 전후 max degree 비교)**가 슈퍼허브 제거 효과를 확인한다.
- **★제외 지목 결정 — 측정 기반(R2 교훈)**: 어느 지목을 뺄지 *추측 말고 측정*했다.
  - 4구 지목: 대 74.7% / **도(도로) 15.6% → 성북 max degree 748 슈퍼허브** / 임 3.3%(forest).
  - **임/전/답은 노드로 유지**: positive 라벨에 임/전/답 **379필지**(산동네 재개발) 겹침 →
    blanket 제외 시 positive 2,728 손실. "건물 있으면 유지" override는 *도로+건물=657 허브*
    재생성으로 역효과(측정 확인) → **off**. 남는 forest 1개(max degree 149)는 2층 GNN에
    영향 제한적이라 수용. size-기반 hub guard는 v1.1.
  - **제외 = 도·천·구·철·제·유·공·수·묘**(linear/water/비건축). 결과: max degree 748→149,
    99% degree ≤13. 비노드 positive(도로 2,315 등)는 `reconcile_labels_to_graph`로 drop
    (구역 폴리곤 안 도로 필지 — 재개발 노드 아님). 라벨 96.26%→100% 그래프 일치.

## 3. ★데이터 가용성 — v1 피처 vs 데이터 대기
지금 가진 것: 연속지적도(geometry·지번·**지목**[A5]), GIS건물(사용승인·구조·연면적·층수).
| 피처 | v1 가능? | 출처 |
|---|---|---|
| 노후도 (시점 t) | ✅ | aging + buildings (R1) |
| 면적 | ✅ | parcel geometry.area |
| 형상(compactness 등) | ✅ | parcel geometry |
| 호수밀도(국지 건물동수/면적) | ✅ | buildings count + 면적 |
| 접도(도로 인접) | ✅ | 지적도 **지목='도'** 필지 인접 여부 → parcels에 A5(지목) 추가 필요 |
| 거리(하천·공원) | △부분 | 지적도 지목='천/공' 필지까지 거리(있으면) |
| 용도지역 | ❌ v1.1 | 용도지역도(별도) |
| 공시지가 | ❌ v1.1 | 개별공시지가(별도) |
| 거리(지하철역) | ❌ v1.1 | 역사 위치 layer |
| 배제레이어 저촉(그린벨트·문화재·군사) | ❌ v1.1 | 완전판 폴리곤 |

→ **v1 피처 = 노후도(시점)·면적·형상·호수밀도·접도 (+가능시 하천/공원 거리) + 그래프 구조.**
데이터 의존 피처(용도지역·공시지가·역거리·배제)는 입수 후 확장(leakage_ablation으로 영향 측정).

★**R9 공정성**: 베이스라인(XGBoost+이웃집계)도 **동일 v1 피처 집합**만 쓴다. 통제 비교는
피처 동일성이 전제 — GNN만 추가 피처를 쓰면 비교가 무의미.

## 4. 함수/파일 분해 (④)
`graph/build.py`
| 함수 | 역할 |
|---|---|
| `build_adjacency(parcels, *, buffer=0) -> edge_index` | 경계 공유 필지쌍 → PyG `[2,E]` LongTensor. STRtree로 근방만. |
| `build_graph(parcels, sigungu_codes) -> Data` | 구 단위 배치로 전역 그래프 + node_index(pnu↔idx) 매핑. 캐싱. |

`graph/features.py`
| 함수 | 역할 |
|---|---|
| `node_features(label_rows, parcels, buildings, *, cfg) -> (X, pnu_index)` | (필지,시점) 행마다 t 기준 피처 벡터. |
| `_buildings_as_of(buildings, t)` | ★보강1: 사용승인일≤t 건물만. **건물 파생 피처(노후도·호수밀도)는 전부 이걸 거친다** — 노후도만 시점필터하면 호수밀도에 t 이후 신축 빌라가 새는 누수(R1). |
| `_aging_feat / _area_feat / _shape_feat / _density_feat / _road_feat` | 피처별. 건물파생(노후도·밀도)은 `_buildings_as_of(t)` 입력. 각 함수에 "왜 후보경계에 의미있나" 주석. |

> ★피처 t-정합: node_features는 *라벨 행의 t*로 노후도를 계산한다. 같은 PNU가 두 t로
> 두 행이면(R5) 두 피처 벡터 — 정적 그래프 위 두 학습예시(키스톤 §2-3).

## 5. 검토했지만 버린 대안 (③)
| 대안 | 기각 |
|---|---|
| PostGIS ST_Touches 구 전체 한 번에 | 메모리 폭발(R10). 배치+STRtree. |
| 시점별 그래프 재구축 | v1은 현재 그래프 1개(분합필 resolve). 시점별 복원 v2. |
| 동적/시계열 GNN | 과한 도구. 정적 스냅샷 + t별 피처(키스톤). |
| 도로 무시하고 strict touches만 | 도로로 갈린 블록 단절 → 작은 버퍼 옵션(config). |
| 가격 피처 절대값 | 시대 불일치 누수(labels §9) → (연도,구) 백분위(데이터 입수 시). |

## 6. ★수검 계획 (규칙9)
- build: 엣지 **대칭**(무향)·자기루프 0·고립노드 비율·degree 분포. ★**수정1 효과 =
  비주거 지목 제외 *전후* max degree 비교**(슈퍼허브 사라지나)·자치구 경계 끊김 확인.
- features: ① 같은 PNU 두 t 행의 노후도가 **실제로 다른지**(R1) ② ★보강1 = **호수밀도도
  t에 따라 다른지**(t 이후 신축 빌라 배제됐나) ③ 결측·범위.
- 통합: 라벨 PNU(주거계)가 전부 그래프 노드에 있나(매핑 누락 0).

## 7. 미해결 / 주의
- 접도/하천·공원 거리 위해 **parcels.py에 A5(지목) 컬럼 추가** 필요(작은 확장).
- 버퍼 폭·호수밀도 반경은 config(매직넘버 금지). 실측으로 정함.
- R3(공간 누수) 버퍼는 eval/spatial_cv의 일이지만, 그래프 degree·수용영역이 버퍼폭을
  정하므로 여기서 layer 수와 함께 메모.
