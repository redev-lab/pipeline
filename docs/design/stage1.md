# 설계 노트 — `rules/stage1.py` (Phase 4, Stage1 요건 룰셋 + 경로 분류)

> 규칙1: 코드 전에 이 노트. 승인 후 구현. CLAUDE.md §2(규칙4·5)·§6 우선.

## 0. 한 문단 요약

필지 클러스터(PNU 집합)를 받아 **법적 정비 요건 충족도를 결정론적으로 점수화**하고
경로(재개발 / 모아타운·소규모정비 / 해당없음)를 분류한다. 학습 아님 — 투명한 루브릭.
모든 임계값은 `legal_thresholds.yaml`에서 읽고(규칙5), 노후·불량은 **사용승인일 기반
근사**임을 출력에 명시(규칙4 정직성). 심장1(GNN/B1)이 *어디를* 후보로 보느냐와 무관하게,
*그 후보가 요건을 충족하는가*를 따로 답하는 단계(진단/예언 분리, §6의 '진단' 쪽).

## 1. 이 모듈이 하는 일

입력 = 필지 클러스터(PNU 집합). 출력 = ① 요건별 지표값·충족여부 ② 경로 분류
③ 근거 문자열(어느 요건을 어떤 수치로 통과/미달). **모델 무관**(R9 정신): B1 점수에서
클러스터를 조립하는 건 infer 단계이고, stage1은 그 클러스터를 *인자로 받는 순수 함수*라
infer 없이 단독 테스트된다.

## 2. ★시점 — 추론용이라 현재 노후도 (R1 무관)

학습 라벨은 as-of-t로 누수를 막았지만(R1), **Stage1은 학습이 아니라 "지금 이 동네가
요건을 충족하나"를 답하는 서비스 추론**이다. 따라서 **현재 시점 노후도**(`old_ratio_*`
current_year)를 쓴다 — 미래정보 누수 개념 자체가 없다(예측이 아니라 현황 판정).

## 3. 요건 지표 — 무엇을 어떻게 계산하나 (데이터 한계 명시)

클러스터의 parcels(면적·지목) + buildings(사용승인일·구조·연면적)에서 결정론 계산:

| 지표 | 계산 | 임계값(config) | ★데이터 한계 |
|---|---|---|---|
| 노후·불량 연면적 비율 | Σ(노후건물 연면적)/Σ(전체 연면적), 현재시점 | housing 0.60 / urban 0.30 | 사용승인일 근사(실제 구조안전 진단 아님) |
| 접도율 | 도로 접한 필지 수 / 전체 필지 수 | ≤0.40 | ★road_abut은 *폭 무관* touches — 조례의 '4m 도로폭' 미반영(v1 근사, 출력 명시) |
| 호수밀도 | 건물 동수 / 클러스터 면적(ha) | ≥60 | ★'동'이지 '호(세대)' 아님 — 집합건물 세대수 미반영(v1 근사) |
| 과소필지 비율 | (면적<과소컷 필지)/전체 | ≥0.40 | 과소컷(예 90㎡)은 config TODO로 조항 확인 |
| 저밀이용 비율 | (용적률<저밀컷 필지)/전체. 용적률=Σgfa/대지면적 | ≥0.50 | 법정용적률 대비가 정석이나 용도지역 데이터 대기 → v1은 절대 FAR 근사 |

★ 모든 한계는 출력 근거 문자열에 한 줄씩 박는다(규칙4·R15 정직성).

## 4. 경로 분류 로직

- **도시정비형 충족** = {노후도≥0.30, 과소필지≥0.40, 저밀≥0.50} 중 **2개 이상**.

## 7. ★수검 계획 (규칙9, 구현 후 — 설계 단계에 미리 박음)

루브릭이 틀렸는지는 **실제 지정구역으로 역검증**한다. ★**기대치(중요)**: 충족률 100%를
합격선으로 잡지 않는다. 100%가 안 나오는 정당한 이유가 최소 셋:
- (a) **옛 구역은 당시 기준/촉진 특례**로 지정 — 현행 룰과 다름(연대 효과).
- (b) **UQ1222 도시정비형은 요건 자체가 다름** → zone_type별 분리 집계 필수.
- (c) **신통 선정은 자체 평가체계** — 법정 요건과 1:1 아님.

→ 수검 출력은 "충족률 X%" 단일 숫자가 **아니라**:
1. **zone_type별·연대별 충족률 표** (UQ1221 주택정비형 vs UQ1222 도시정비형, 지정연도 구간별).
2. **미달 구역 전수 목록** — 각 미달이 *어느 지표에서 얼마나* 미달인지(노후도 0.55<0.60 등).
3. ★**판정 기준**: **주택정비형(UQ1221) 옛 구역 = 우리 룰의 본진**. 거기서 다수 통과하면 룰
   합격. 본진에서도 대거 미달이면 룰 또는 지표 계산이 틀린 것. 미달이 **특정 유형에 몰리면
   그게 발견**(완공/신축혼재/연대특례), **무작위로 흩어지면 근사치 오차**.
4. **명백한 negative 미달**: 4구 내 신축파생 클러스터(reliable_negative) 1~2개 → "해당없음"
   확인(역삼은 강남 ingest 필요 → v1.1 데모, HANDOFF 메모: Phase 8 데모 3종이 마포·강남
   추론용 구 로드를 어차피 요구).
5. **경로 분리**: 광역 미달이나 노후한 클러스터가 "모아타운"으로 빠지는지 1건 확인.
