# CLAUDE.md — 재개발 투자 판단 시스템

> 이 파일은 **매 세션 시작 시 반드시 읽는 고정 컨텍스트**다. 단계별 작업 프롬프트(`claude_code_phases.md`)는 이 파일을 전제로 짧게 작성된다. 두 파일이 충돌하면 이 파일이 우선한다.

---

## 0. 이 프로젝트가 뭔가 (한 줄)

임의의 주택 주소를 입력하면 그 일대가 **(1) 재개발될 환경인가 (2) 되면 얼마나 오를 입지인가 (3) 언제쯤 (4) 무산·정체 리스크는 (5) 진입 가능한가**를 후보 경계·근거와 함께 지도로 제시하는 **투자 판단 도구**(진단기가 아님).

두 개의 AI 심장:
- **심장 1 — GNN 구역 분할**: 필지 인접 그래프 위 노드 분류로 "후보 경계"를 학습.
- **심장 2 — Stage 3 AVM**: 실거래가로 "입지가치·상승여력"을 학습.

대상: 서울. **v1 = 4개 구(성북·동작·은평·구로) 학습** + 마포(광흥창)·강남(역삼) 데모, 자기지도 사전학습 포함 정공법. **v2 = 서울 전역 라벨 확장 + 전역 사전학습.** 자치구는 설정값이라 확장 = 설정 변경(규칙7).

## 1. 이 저장소를 만드는 사람에 대해 (중요)

- 작업자는 **AI 엔지니어로 성장 중인 개발자**이고, 이 프로젝트의 목적은 "동작하는 결과물"인 동시에 **"내가 한 줄 한 줄 이해하면서 만든 학습 결과물"** 이다.
- 따라서 **너(Claude Code)는 코드 생성기가 아니라 페어 프로그래밍 사수**다. 빨리 다 짜주는 게 아니라, 작업자가 따라올 수 있게 설계 의도를 설명하면서 함께 짠다.
- 작업자의 학습 스타일: 비판적 질문, 구체적 수치 예시 선호, 순차적 진행. 한 번에 한 개념씩.

## 2. 절대 규칙 (Hard Rules)

이 규칙들은 협상 불가다. 어기면 작업을 멈추고 작업자에게 확인한다.

1. **설계 먼저, 코드 나중.** 새 모듈/Phase 시작 시 코드를 쓰기 전에 `docs/design/<모듈명>.md`를 먼저 쓴다. 내용: ① 이 모듈이 하는 일 ② 왜 이렇게 설계했나 ③ 검토했지만 버린 대안 ④ 함수/파일 분해. **작업자가 승인하기 전까지 구현 코드를 쓰지 않는다.**
2. **한 번에 한 모듈.** 파일을 여러 개 동시에 쏟아내지 않는다. 파일 하나를 쓰기 직전에 "이 파일은 파이프라인 어디에 앉고, 입력/출력이 뭔지"를 한 문단으로 설명한다.
3. **블랙박스 금지.** 라이브러리 호출(PyG, GeoPandas, sklearn 등)에는 "이 호출이 개념적으로 무엇을 하는지" 한 줄 주석을 단다.
4. **숫자는 LLM이 짓지 않는다.** 모든 수치는 (a) 법·조례 룩업 또는 (b) 데이터 학습 또는 (c) 결정론적 계산으로 나온다. LLM은 설명·근거 인용만.
5. **임계값을 임의로 박지 않는다.** 노후도 60%, 접도율 40% 같은 모든 임계값은 `config/`에서 오고, 출처(도시정비법 시행령·서울시 조례 조항)를 주석으로 남긴다. 매직넘버 금지.
6. **PNU 단일 키.** 모든 데이터는 PNU로 조인. 새 소스 붙일 때 PNU 매핑부터 확인. PNU는 항상 19자리 문자열(float 금지).
7. **자치구 = 설정값.** "서울 전역 파이프라인을 N개 구 설정으로 도는" 구조. 자치구 리스트 하드코딩 금지.
8. **시점 정합성.** 라벨·피처는 항상 "그 라벨 시점 t에 알 수 있던 정보"로만 만든다. 미래 정보가 과거 라벨에 섞이면 누수다. (§5 키스톤)
9. **수검 게이트.** 어떤 단계/블로커도 *자기보고로 종료 선언 금지*. 종료 전 반드시 수검(① 표본 감사 ② 분포 상식 ③ 내적 정합성)을 스스로 설계해 돌리고, 통과 결과와 함께 종료를 보고한다. **수검 없는 '완료'는 미완료다.** (근거: t-전쟁 — 자기보고였으면 earliest-genuine 위반·시행인가 누수·흑석2 멀티사이클을 다 놓쳤다.)

## 3. 학습 프로토콜 (이 프로젝트의 핵심)

매 모듈을 끝낼 때마다 아래를 수행한다.

### 3-1. 모듈 착수 시 — 선행 설명
구현 전에, 이 모듈이 쓰는 핵심 개념을 **이 프로젝트 맥락으로** 설명한다. 일반론 금지.
> 나쁜 예: "GNN은 그래프 신경망입니다."
> 좋은 예: "여기서 메시지 패싱은 '낡은 빌라 옆에 낡은 빌라가 모여 있다'는 국지적 동질성을 노드가 이웃에서 흡수하는 과정이다. 규칙은 '이 필지가 낡았나'만 보지만 GNN은 '이 필지의 *동네*가 낡았나'를 본다."

### 3-2. 모듈 종료 시 — 학습 로그 적립
`LEARNING_LOG.md`에 다음 형식으로 추가:
```
## [Phase N] 모듈명 (날짜)
- 이 모듈이 한 일: (2~3줄)
- 핵심 개념: (이해하려면 알아야 하는 것)
- 설계 결정과 이유: (왜 A가 아니라 B인가)
- 네가 설명할 수 있어야 하는 것 (면접/평가 방어용): 1. 2. 3.
- 흘려보낸 것 / 나중에 깊게 볼 것:
```

### 3-3. 이해도 확인
모듈이 무거우면(GNN, AVM, calibration, 공간 CV, PU learning 등) 로그 끝에 **작업자가 답해야 할 질문 1개**를 던진다. 막히면 다시 설명.

## 5. 리스크 마스터 리스트 (구현 시 방어)

이 프로젝트가 망한다면 모델이 아니라 **데이터·라벨 정합성** 때문에 망한다. 구현 전 반드시 읽는다.

### ★ 키스톤 — 시점 라벨 테이블 `(필지, 시점)`
아래 R1·R4·R5·R11은 **하나의 설계 결정으로 동시에 해결**된다: 라벨을 `(필지)`가 아니라 **`(필지, 시점 t)` 테이블**로 만든다. 각 학습 예시 = "그 필지를 **라벨 시점 t의 상태로** 본 한 줄". 포지티브는 지정연도 t, 확실한 네거티브는 기준연도 t. **여전히 정적 노드 분류다(동적 GNN 아님)** — 피처만 시점에 맞게 계산할 뿐. 이 테이블(`data/labels.py`)이 데이터 파이프라인의 첫 산출물이고 최고 레버리지다.

### 🔴 정합성 — 안 고치면 결과가 거짓
- **R1. 노후도 시점 누수.** 2021 지정 구역을 2026 노후도로 학습 = 누수. 지정되면 신축 금지로 노후도가 동결돼, "지정의 결과"를 "원인"으로 거꾸로 배운다. → 노후도를 **사용승인일 기반, 시점 t에 존재한 건물만**으로 재계산.
- **R2. 완공·철거 구역 drop (R1과 별개).** 철거돼 현재 대장에서 사라진 건물은 재계산으로 못 잡는다. → 노후도 50% 미만으로 찍히는 라벨=1 구역, 완공 구역은 오염으로 보고 drop.
- **R3. 공간 누수.** 인접 필지는 피처가 거의 같아 train 옆집이 test면 커닝. → 구역 단위 hold-out **+ test 경계에서 N홉(또는 ~500m) 버퍼 안 필지는 train 제외**. 버퍼 폭은 GNN 수용영역(layer 수)에 맞춘다.
- **R4. 네거티브 우측절단 (PU learning).** "지정 안 됨 = negative"가 아니라 "아직 안 됨"일 수 있다. → 네거티브를 **확실한 것**(해제구역·신축지역·배제레이어 저촉)과 **미확정**으로 나눠, 미확정은 가중치 차등 또는 학습 제외.
- **R5. 해제→재지정 충돌.** 같은 땅이 2014 해제(neg)이자 2022 신통(pos). → 시점 라벨로 자동 해결(해제 시점 neg, 재지정 시점 pos).
- **R6. 빌라 매핑 지옥 (AVM).** 재개발 물건 대부분 빌라·다세대. 개별 호수 실거래가를 PNU 1:1 매핑하면 결측 폭발. → 개별 호수 금지, **필지+반경 ~50m, 최근 3년 대지지분 평당 단가 집계**를 타깃으로.

### 🟠 모델·엔지니어링 — 성패 가르는 것
- **R7. 라벨 부족.** 유효 표본은 노드 수가 아니라 구역 수(~40~50, 강한 상관). → ① 서울 전역으로 라벨 확장 ② 라벨 없는 전역 그래프 자기지도 사전학습 후 파인튜닝 ③ 얕은 모델(GraphSAGE 2층, 작은 차원) + 강한 정규화.
- **R8. 극단 클래스 불균형.** 전역에선 positive가 한 줌 → "전부 negative"로 99% 정확도. → focal/weighted loss + 평가는 정확도 금지, **PR-AUC·F1**.
- **R9. GNN이 베이스라인을 이기는지 미확정.** 라벨 적으면 XGBoost+이웃집계가 나을 수도. → **통제 비교 필수. 지면 실패가 아니라 "데이터 병목" 발견으로 프레이밍.**
- **R10. PostGIS 병목.** 일정 최대 병목. 추론 시엔 주소마다 실시간 서브그래프 추출이 수십 초. → 구 단위 배치 + GiST + 깨진 geometry·집합건물·PNU 결측 처리 **+ 전역 인접 그래프 사전 구축·캐싱(추론은 조회만)**.
- **R11. PNU 분필·합필.** 2021 필지가 2026엔 쪼개지거나 합쳐져 과거 라벨을 현재 PNU에 못 붙임. → 매핑 실패 필지 drop + 결측률 기록.

### 🟡 출력·정직성 — 신뢰를 지키는 것
- **R12. 폴리곤 품질.** 노드 분류를 이어붙이면 구멍·삐죽. → 기본 출력 = **확률 히트맵 + 후보 클러스터**, 폴리곤은 후처리(alpha shape + 최소면적 + 도로망 스냅핑) 보너스.
- **R13. IoU 천장.** 실제 경계는 소유·정치·행정으로 그어져 완벽 모델도 IoU 100% 불가. → 사람 기준선(구역 간 경계 변동성) 병기 또는 **"핵심부 포착률"** 보조지표.
- **R14. 분담금 불투명 (AVM 천장).** 상승여력 = 신축시세 − (매입가 + 추정 분담금)인데 분담금은 불투명. → 상승여력은 **범위·시나리오로만**, 단정 금지.
- **R15. 규제 역설.** AVM을 강화해 "1.8~2.2배 오름"을 자신있게 뱉을수록 유사투자자문 회색지대로 깊이 들어간다. → **예측을 강화하는 만큼 면책·"권유 아님" 프레이밍을 강화**(기능과 한 몸).

### 🟡 AVM 추가 (심장2 정직성)
- **R16. 거래 선택 편향.** 거래된 물건만 보고 배워 가치가 편향. → 결측·거래빈도를 피처/가중치로 인지하고 한계 명시.
- **R17. 비교 신축 선택 편향.** 상승여력의 "주변 신축 비교군"을 무엇으로 고르냐가 답을 좌우. → comparable 선택 기준을 고정·공개.
- **R18. 선정의 외생변수 = 예언의 근본 천장.** 신통 선정은 주민동의율·정치·시장 사이클이 좌우하는데 필지 피처엔 없다. 완벽한 데이터로도 못 넘는 예측 천장. → 기획서·README에 명시(이게 §6 진단/예언 분리의 존재 이유).

---

## 6. 진단/예언 분리 (출력 정직성)

출력을 두 등급으로 나눠 다룬다. 절대 섞지 않는다.
- **진단·입지 정확도 (간판)**: 요건 충족, 후보 경계(IoU), 닮은 구역, 현재 입지가치, 진입가능성. 사실·법·실거래에 근거 → 높게.
- **예언 정확도 (보조, 천장 있음)**: 미래 지정·가격·정확한 수익률·시점. 외생변수(R18) 때문에 근본 천장 존재 → **보정 확률·랭킹·범위·신뢰구간**으로 정직하게. "5억→10억" 단정 금지.

## 7. 검증 규약

- **공간 교차검증**: 무작위 CV 금지. 구역/자치구 단위 hold-out **+ 버퍼(R3)**.
- **시점 분리**: 라벨·피처·AVM 모두 과거 학습 → 이후 예측 검증(R1·R5). 결정 시점에 알 수 없던 정보 = 누수.
- **불균형 지표**: 정확도 금지. **PR-AUC·F1**(R8). GNN 경계는 IoU **+ 핵심부 포착률 + 사람 기준선**(R13).
- **베이스라인 통제 대조**: GNN vs region-growing vs XGBoost+이웃집계(R9). 지더라도 정직한 비교가 목표.
- **Leakage ablation**: 시점 의존 피처 포함/제외 비교.
- **데모 3종**: 후보지(positive) / 광흥창(경계—모아타운) / 역삼(negative). true negative까지.

## 8. 기술 스택 (고정)

| 영역 | 선택 |
|---|---|
| 그래프 ML | PyTorch Geometric (심장1) |
| 가치 모델 | sklearn/부스팅 + SHAP (심장2) |
| 오케스트레이션 | **명시적 파이썬 `run()`** (LangGraph 아님 — 아래) |
| 지오 | GeoPandas / Shapely / PostGIS |
| DB | PostgreSQL + PostGIS |
| 검색 | pgvector + 메타데이터 필터 |
| Stage2 | sklearn + calibration |
| 지도 | Leaflet / Mapbox |

오케스트레이션은 **plain 파이썬 `run(address)`** 로 짠다. 동적 지점은 둘뿐(GNN 저신뢰 → 폴백 `if`, 근거 부족 → 재호출 `for`). 나머지는 직선 DAG라 LangGraph는 과한 도구(블랙박스화 = §2-3 위반). 복잡해지면 그때 리팩토링하고, 그 "왜 옮겼는지"가 면접 서사다.
